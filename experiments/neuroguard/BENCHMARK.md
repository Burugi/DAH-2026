# NeuroGuard 성능 벤치마크 — 시나리오 × 방어

**무엇을 재나:** 23개 공격 시나리오(`configs/attack_scenarios.yaml`)를 **무방어 / 코디네이터 / 통합방어 /
통합방어(현실 탐지)** 로 같은 seed에서 돌려 **가용성 곱셈종합**(= `mean(1−점령, 1−AUC, 탐지F1) × 가용성`,
↑좋음)을 비교. 재현: `runners/exp_matrix.py` → `results/summary_matrix.csv`.

## 간단한 해석
1. **통합방어가 코디네이터를 거의 전 시나리오에서 개선**(평균 0.736→0.779, worst 0.354→0.427). 가장 크게 오르는 축은 **재밍/가용성**(0.599→0.744)과 **연결성**(A-CONN 0.354→0.482) — de-jam·relay·분산자율이 약점 축을 메움.
2. **진짜 하드 클래스 = 연결성**(A4·A-CONN·A18) — 통합방어로도 0.43~0.59로 최하위. (동적 재구성 v2로 별도 해결: `docs/추가연구_특이점과한계돌파` 참고.)
3. **현실 탐지(오라클 해제)에선 이점이 상쇄**(0.779→0.731). 특히 **탐지저하 공격(A11/A15/A21)이 0.819→0.699(−15%)** — 오라클 가정이 이 클래스를 가장 크게 부풀렸음. → 본선 전이 시 탐지 노이즈·신념 추정층이 필수.
4. 여러 시나리오가 **동일 수치**(A1/A5/A9, A3/A6/A7/A13, A11/A15/A21) — 23개 라벨이 소수 sim 기제로 수렴함(허위정밀 주의).

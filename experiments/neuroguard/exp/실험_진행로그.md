## 설계 — 중앙집중 단일에이전트 (공정 비교용)
- **관측:** 감염 플래그 벡터(n=18) + 정규화 step → shape (19,).
- **행동:** Discrete(19). 0=전체 Sleep, k=드론 (k-1)을 RemoveOtherSessions로 정리(감염 시).
- **M1 휴리스틱:** 최저 index 감염 드론을 정리(스크립트).
- **M2 PPO:** 어느 드론을 정리할지 학습.
- **M3 하이브리드:** 휴리스틱 action-mask(감염 드론만 유효) + MaskablePPO → 무효행동 사전 제거.
- **보상:** CybORG 벤치마크 step 보상(공정). 감염수는 별도 측정.
- red: RedDroneWorm (start=3, spawn=0.20) 고정, defended config 기반.

### 단일에이전트(중앙집중) RL 추가 완료 — 단일 vs 멀티 vs 휴리스틱
`single_agent_rl.py`: 중앙집중 글로벌-Q 학습(전역 감염비율 상태 → 한 결정을 전체 드론에 적용) vs 코드팀 멀티 공유-Q vs 휴리스틱. **모두 rule-red 상대, held-out seeds 1000~1004.**

| 방어자 | 최종 감염(낮을수록↑) | 보상(높을수록↑) |
|---|:---:|:---:|
| **heuristic (multi)** | **0.356** | **−453** |
| RL-multi (드론별 Q) | 0.867 | −534 |
| RL-single (중앙 Q) | 0.756 | −594 |

**해석 (사용자 질문 직답):**
- **휴리스틱이 두 지표 모두 압도** (감염 0.36, 보상 −453).
- **"멀티가 무조건 유리"는 거짓** — 봉쇄(감염)는 RL-single(0.76)이 RL-multi(0.87)보다 **오히려 나음**, 보상은 RL-multi가 나음 → 둘은 trade-off, **둘 다 휴리스틱에 패배**.
- 이유: 약한 tabular-Q는 per-drone 정보(멀티)를 더 나은 봉쇄로 못 살림. 휴리스틱의 "자기감염→제거 / 의심→탈환" 손코딩이 더 정확. (CAGE CC4 교훈 = 휴리스틱>MARL 재확인)
- ⚠️ 한계: 약한 tabular-Q 기준. 강한 deep RL이면 달라질 수 있음 → 보고서 7.4 로드맵(PPO/GNN).

## 하이브리드(휴리스틱+학습) 실측 — 사용자 질문 직답
"에이전틱이 휴리스틱보다 안 나올 수 있나? 휴리스틱을 에이전틱에 넣고 추가 상황만 고려하면?" → **정확한 통찰. 실측으로 증명.**
- `hybrid_rl.py`: 휴리스틱 코어(자기감염→제거) 고정 + **그 외 상황만** 학습이 합리적 방어(재장악/차단/디코이=PROACTIVE [4,6,8]) 중 선택. rule-red 상대, held-out seeds.

| 방어 | 최종 점령(↓) | 보상 |
|---|:---:|:---:|
| 휴리스틱(멀티) | 0.356 | −453 |
| RL-멀티 | 0.867 | −534 |
| RL-단일 | 0.756 | −594 |
| **하이브리드** | **0.311** | −508 |

- **1차 시도 실패(0.522):** RL이 휴리스틱의 핵심 retake까지 자유롭게 덮어써 강점 상실 → 교훈: 학습이 좋은 규칙을 *덮어쓰면* 안 됨.
- **수정(floor 보존, PROACTIVE 제한) → 0.311:** 휴리스틱(0.356) **능가**. 단 보상 −508<−453 = 차단/디코이 비용 trade-off.
- **결론:** 순수 학습 < 휴리스틱 < 하이브리드(floor+학습). 학습은 from-scratch가 아니라 **hybrid-first**일 때 가치. (C3·B20 정량 근거)
- 증거: `fig10_hybrid.png`, `summary_hybrid.csv`. 보고서 보강 → `DAH2026_docs\예선_결과섹션_보강.md`.

## 딥 PPO 로드맵 실측 (심화)
`drone_ppo.py`: SB3 PPO 학습(40k step) — PPO-순수 vs PPO-하이브리드(휴리스틱 floor+딥). `fig11`.

| 방어 | 점령(↓) |
|---|:---:|
| 휴리스틱(멀티) | 0.356 |
| RL-멀티(tabular) | 0.867 |
| **드론별 하이브리드(tabular)** | **0.311** |
| PPO-순수(딥,중앙) | 0.867 |
| PPO-하이브리드(딥,중앙) | 0.511 |

- **딥 PPO-순수 = tabular RL(둘 다 0.867)** → 학습기 키워도 from-scratch는 휴리스틱 못 따라잡음. **용량 문제 아님, 접근법 문제.**
- **2축 교훈:** ① 휴리스틱 floor가 최대 레버(없으면 0.87) ② per-agent 세분화(하이브리드 안 0.311<0.511). → "멀티 유리"는 *floor 위 per-agent*일 때 성립.
- 본선 주력 = **드론별 하이브리드**. 딥/GNN은 그 위 보강재.
- 증거: `fig11_ppo_roadmap.png`, `summary_ppo.csv`. 보고서 §6.3.5로 정리.

> **관련 문서:** `DAH2026_전략문서.md`, `시나리오_아이디에이션.md`, `CAGE_Challenge_리서치.md`

## 1. UAV / UGV 개념

- **UAV (Unmanned Aerial Vehicle, 무인항공기):** 흔히 드론. 임무 = 정찰·감시(ISR), 통신중계, 타격, 자폭(loitering munition) 등.
- **UGV (Unmanned Ground Vehicle, 무인지상차량):** 정찰, 물자수송, 폭발물처리(EOD), 전투 지원 등.
- **핵심 관점:** 무인체계는 단일 기체가 아니라 **[플랫폼 + 지상통제(GCS) + 통신링크(C2) + 페이로드]** 로 이루어진 *system of systems*. → **공격면도 이 4축에 분산**된다.
- **군용 UAV 분류 (미 DoD Group):**

| Group | 무게(MGTW) | 운용고도 | 속도 | 예 |
|-------|------|----------|------|----|
| 1 | <9kg (<20lb) | <1,200ft AGL | <100kt | 소형 정찰 드론 |
| 2 | 9~25kg | <3,500ft AGL | <250kt | 중소형 |
| 3 | <600kg | <18,000ft MSL | <250kt | 전술 UAV |
| 4 | >600kg | <18,000ft MSL | 제한없음 | MALE급(중고도 장기체공) |
| 5 | >600kg | >18,000ft MSL | 제한없음 | HALE급(예: 글로벌호크) |

> 무게·고도·속도 **셋 중 하나라도 상위면 상위 그룹**. G4/G5 핵심 구분자=18,000ft MSL. (CRS IF12797 — 출처_근거 §D-3)

## 2. UAV 시스템 구성 (공격면 이해의 핵심)

| 구성요소 | 설명 | 대표 |
|----------|------|------|
| **비행제어컴퓨터(FCU)** | 자세·비행 제어 하드웨어 | Pixhawk |
| **오토파일럿 SW** | 비행제어 펌웨어/스택 | **PX4**, **ArduPilot** (오픈소스, 광범위 사용) |
| **지상통제장비(GCS)** | 임무계획·조종·모니터링 | QGroundControl, Mission Planner |
| **통신링크(C2/Datalink)** | 기체↔GCS 데이터 | RF(텔레메트리), 셀룰러, **SATCOM(위성)** |
| **통신 프로토콜** | 명령/텔레메트리 메시징 | **MAVLink** |
| **항법센서** | 위치·자세 | **GNSS(GPS)**, IMU(자이로/가속도), 기압계, 지자기, (고급)RTK |
| **페이로드** | 임무 장비 | EO/IR 카메라, SAR, 무장 |
| **컴패니언 컴퓨터** | AI·영상처리·자율 | Jetson, RPi + **ROS/ROS2** |
| **임베디드 기반** | 실시간 운영체제·펌웨어 | RTOS(PX4는 NuttX), 부트로더, OTA |

> 🔑 **우리 팀 임베디드 강점이 꽂히는 지점:** FCU·오토파일럿·RTOS·펌웨어·센서 = 임베디드 영역. 여기서 공격면/방어를 *현실적으로* 설계할 수 있다는 게 차별점.

## 3. UGV 시스템 구성

- 기본 골격은 UAV와 유사(제어컴퓨터·통신·센서·자율SW)하나 **지상 주행**에 특화:
  - 구동/조향 액추에이터, **LiDAR·카메라·초음파·레이더** 센서, **SLAM/자율주행 스택**, **ROS2** 중심.
  - 통신: RF/메시 네트워크/셀룰러 + **원격조종(teleoperation)**.
- **특징:** 센서 의존도가 매우 높음 → **센서 스푸핑(LiDAR/카메라 기만)** 위협이 UAV보다 큼. 저속이지만 물리 충돌·탈취 리스크.

## 4. 통신 구조 & 프로토콜 (대회 "위성망 모사" 환경 직결)

### 4.1 MAVLink (가장 중요)
- UAV 표준 경량 메시징 프로토콜(PX4·ArduPilot·QGC 공용). `HEARTBEAT`, `COMMAND_LONG`, `MISSION_ITEM` 등.
- **보안 약점:** **MAVLink 1.0은 인증·암호화 없음(평문)** → 명령 위조·주입·스니핑·재전송에 취약. MAVLink 2.0은 **메시지 서명**(SHA-256 기반 48비트 — 표준 HMAC 아닌 *secret-prefix MAC*; **인증·무결성만 제공, 암호화(기밀성) 없음 → 페이로드는 여전히 평문**)을 지원하나 **기본 OFF·키분배 부재로 실배치 미적용 다수**. (출처_근거 §D-3)
- → ① 공격(명령 주입)과 ② 방어(무결성/인증) 모두의 핵심 무대.

### 4.2 위성망 (SATCOM) — 대회 환경의 정체성
- 위성 경유 C2 링크. **특성: 높은 지연(latency)·간헐적 단절·대역폭 제한·광역 도달.**
- 보안 위협: 링크 암호화 여부, 재밍, 스푸핑, **재전송(replay)**, MITM.
- 🔑 **대회가 "위성망 모사"** → **지연·단절을 악용하는 공격**(예: 단절 구간에 자율 악성행동)과 **단절에 강인한 방어**(자율 페일세이프)가 차별 포인트.

### 4.3 RF 일반 & ROS2
- RF: **재밍(jamming, DoS)**, **미코닝(meaconing, 재방송 기만)**, 프로토콜 익스플로잇.
- **ROS / ROS2 + DDS:** ROS1은 기본 보안 전무. ROS2는 DDS 미들웨어 + **SROS2(보안 확장)**. 미적용 시 **DDS 디스커버리 악용·토픽 인젝션** 위협.

## 5. 항법(Navigation) & GPS/GNSS 보안

- **GNSS 종류:** GPS(미)·GLONASS(러)·Galileo(EU)·BeiDou(중). **민간 신호는 비암호화** → 스푸핑/재밍 취약. 군용 **M-code/P(Y)** 는 암호화.
- **재밍 vs 스푸핑:**
  - **재밍(Jamming):** 신호 교란으로 측위 불능(DoS).
  - **스푸핑(Spoofing):** 가짜 GNSS 신호로 **위치·시간을 기만** → 경로 이탈·지오펜스 우회·탈취(hijack). 더 은밀하고 위험.
- **INS/IMU:** 관성항법. GNSS 두절 시 **추측항법(dead reckoning)**. 보통 **센서 퓨전(칼만필터)** 으로 GNSS+INS 결합.
- **방어 기법:** RAIM(수신기 자율 무결성 감시), 다중 GNSS 교차검증, **INS-GNSS 일관성 체크**, 안테나 어레이(신호 도래각 분석), 신호 품질/파워 모니터링.

## 6. 방산 사이버보안 위협 Landscape (공격면 종합)

| 계층 | 공격면 | 대표 위협 |
|------|--------|-----------|
| **공급망** | 펌웨어·부품 | 하드웨어/펌웨어 트로이목마, 악성 OTA |
| **통신링크** | MAVLink·RF·SATCOM | 명령 주입·위조, 재밍, MITM, 재전송 |
| **항법** | GNSS | GPS 스푸핑/재밍 |
| **센서** | LiDAR·카메라·IMU | 센서 스푸핑/기만 |
| **SW/펌웨어** | 오토파일럿·부트로더 | 취약점 익스플로잇, OTA 변조 |
| **GCS** | 지상국 PC | 일반 IT 침투(피싱·취약점) → C2 장악 |
| **컴패니언/ROS2** | 미들웨어 | DDS 인젝션, 노드 탈취 |
| **AI/ML** | 인식·자율 모델 | 적대적 예제(adversarial), 모델 포이즈닝 |

> STRIDE로 보면: **S**poofing(GPS/MAVLink), **T**ampering(펌웨어/명령), **R**epudiation, **I**nfo-leak(스니핑), **D**oS(재밍), **E**levation(권한상승).

## 7. 관련 표준·프레임워크 (보고서 인용용)

- **MITRE ATT&CK (Enterprise):** 사이버 공격 TTP 분류 체계. → **① 공격 시나리오를 단계별 기법 ID로 매핑**(예: `T1190` 외부 노출 서비스 익스플로잇, `T1210` 원격 서비스 익스플로잇, `T1565` 데이터 조작, `T1498/T1499` DoS, `T1489` 서비스 중단). 전문성 점수↑.
- **MITRE ATT&CK for ICS:** 산업제어/사이버물리(CPS)용. UAV/UGV 같은 CPS에 부분 적용. ⚠️ **GPS/센서 신호 스푸핑은 enterprise/ICS ATT&CK에 깔끔히 매핑 안 됨** — 구 `T0856`(Spoof Reporting Message)은 `T1692.002`로 재편됐고 이는 *텔레메트리 보고메시지* 위조이지 GPS 신호가 아님. 효과 최근접=`T0832 Manipulation of View`. (출처_근거 §D-3)
- **STRIDE:** 위협 도출 프레임워크(체계적으로 빠짐없이).
- *(참고만)* 항공 사이버보안 DO-326A/ED-202A, 국방 RMF 등 — 대회 보고서엔 *언급 수준*이면 충분.

## 8. 방산 도메인의 특수성 (→ 설계 철학에 직접 반영)

1. **안전성·가용성 최우선:** 무기체계는 임의로 다운되면 안 됨 → 대회 **`×가용성` 채점과 직결**. 방어가 과해서 기능을 죽이면 실패.
2. **실시간성(real-time):** 비행제어 루프는 ms 단위, 임베디드 RTOS. → AI 에이전트도 **저지연·경량**이어야.
3. **자원 제약:** 임베디드 CPU/메모리/전력 제한 → **경량 탐지·휴리스틱**이 현실적(무거운 딥모델 한계).
4. **공급망 신뢰:** 부품·펌웨어 출처 검증 필수(공급망 공격이 실질 위협).
5. **물리-사이버 융합(CPS):** 사이버 공격이 **물리적 결과**(추락·충돌·임무 실패)로 직결.
6. **적대적 운용환경:** 통신 단절·지연·전자전(EW) 상존 → **자율성·강인성** 요구.

> 🔑 이 6가지가 그대로 우리 설계 원칙: **"가용성 보존 + 경량/실시간 + 자율 강인성"**. CAGE의 가용성 중심 보상과도 일치.

## 9. 핵심 용어집 (Glossary)

| 용어 | 뜻 |
|------|----|
| **MAVLink** | UAV 표준 통신 프로토콜(명령/텔레메트리) |
| **GCS** | Ground Control Station, 지상통제장비 |
| **FCU** | Flight Control Unit, 비행제어컴퓨터 |
| **PX4 / ArduPilot** | 대표 오픈소스 오토파일럿 SW |
| **GNSS / GPS** | 위성항법시스템 / 그 대표 |
| **INS / IMU** | 관성항법 / 관성측정장치 |
| **RTK** | 정밀 GPS 보정(cm급) |
| **SATCOM** | 위성통신 |
| **ROS2 / DDS** | 로봇 미들웨어 / 그 통신 표준 |
| **SROS2** | ROS2 보안 확장 |
| **RAIM** | 수신기 자율 무결성 감시(GNSS 이상탐지) |
| **EW** | Electronic Warfare, 전자전 |

| **CPS** | Cyber-Physical System, 사이버물리시스템 |
| **MANET** | 이동 애드혹 네트워크(군집 통신) |

| **ISR** | 정보·감시·정찰 |

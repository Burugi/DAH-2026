# DAH 2026 — 확장 아이디어 리서치 (창의성·견고함 축, 2023~2026)

> 기존 카탈로그(공격 A1~A17 / 방어 B1~B18) **위에서** 더 깊이·새로운 아이디어를 1차 출처로 발굴.
> 진행: ✅ A축(공격 창의성) / ✅ B축(방어 견고함) — **둘 다 완료**
> ⚠️ 검증: Incalmo만 원문 확인. 나머지는 검색 스니펫 다중 교차확인(일부 PDF 403) → 정량 수치 인용 시 원문 재확인 권장.

## A축 — 공격 창의성 (신규 N1~N10) ✅

### 테마 1: 공격 체인/경로 자동탐색
- **N1. Incalmo ★** — LLM이 *선언적 고수준 태스크*("pivot to X / dump creds / exfiltrate")만 내리면 추상화 계층이 실제 도구로 번역 + **attack-graph가 연쇄 경로 추천**. **40개 환경 중 37개 성공(SOTA 3/40), 12~54분·$15 미만.** (arXiv:2501.16466 *원문 확인*) → **우리 Red 행동집합(Recon→…→Impact) "무한 조합"의 학술 근거**, A14 보강.
- **N2. RL/LLM 자동 공격경로** — RL이 scan→exploit→pivot을 학습해 최적 침투경로 탐색; PENTEST-AI는 ATT&CK+GPT-4로 attack graph를 수시간→수초. (arXiv:2507.02969 ; github Kim-Hammar/awesome-rl-for-cybersecurity) → Red도 RL 학습형(Red/Blue 대칭) 근거.
- **N3. xOffense/AutoPentest** — Recon/Vuln/Exploit 에이전트 **역할분담 멀티에이전트 레드팀** + 막힌 경로 자동 우회(집요함의 알고리즘화). (arXiv:2509.13021 ; 2505.10321) → **신규: 우리 Red를 멀티에이전트로 확장**(C9 계층MARL의 공격 측 대칭).

### 테마 2: 적응형·은밀 공격
- **N4. Raven ★(드론군집 직결)** — **단 1회 false data injection**으로 군집 항법 붕괴하되 **센서 노이즈 한도 내 + 시공간 일관성 유지**로 탐지 회피. 목표 5종(교착/지연/몰이/로봇충돌/장애물충돌), ORCA·GLAS서 검증. (USENIX Security 2025, yeke; PDF 403·스니펫) → **A6·A10을 '탐지회피형'으로 격상**; "왜 임계 탐지만으론 부족한가"의 직접 근거(B1·EX2 시험대).
- **N5. SwarmFuzz** — **1대만 GPS 스푸핑→군집 동역학 타고 연쇄 충돌**(Swarm Propagation). 그래프+gradient 최적화, 무작위 대비 10배·3배 빠름, 클수록 취약. (DSN 2023, ieee 10202617 ; github DependableSystemsLab/SwarmFuzz) → **카탈로그 'SwarmFuzz' 1차 출처 확정**, A3·A14 전파 메커니즘, B5 존재이유.
- **N6. RL 탐지회피(low-and-slow 자동학습)** — 공격 RL이 *"덜 충격적·은밀한 행동 먼저(warm-up)"* 보상으로 APT식 회피 자동 학습; cGAN으로 IDS 우회 트래픽. (arXiv:2602.19582 ; 2506.21142) → **우리 Red 보상에 '은밀성 항' 추가** 근거, B1·B14의 적대 상대.
- **N7. Sensor Deprivation** — 주입이 아니라 **센서를 선택적 박탈**해 융합추정기 은밀 조종(흔적 더 적음). (arXiv:2410.11131) → **A6 신규 변형(주입형↔박탈형)**, EX2 사각지대.

### 테마 3: 메타 공격 (방어 AI 자체)
- **N8. VLM/VLA 프롬프트 인젝션 ★** — 자율체계 시각-언어 모듈에 **이미지 속 스테가노 은닉 명령**으로 정렬 붕괴 → 인지·행동 통째 탈취. SOTA VLM 4종이 594공격에 취약. (arXiv:2507.22304 ; 2601.17383 ; 2506.03350) → **카탈로그 'VLM 프롬프트 인젝션' 1차 출처 확정**; A11·A16 인지층 확장. **우리 LLM 하이브리드(C12 등) 자체가 공격면** = 메타 논점.
- **N9. 상태기반 스텔스 백도어(SSD)** — UAV 경로계획 신경망에 **특정 비행상태 트리거 시만 발동**하는 백도어(평시 정상). (arXiv:2502.20178) → A1·A9·A11 융합 신규; EX3(SLSA)·B8(attestation) 모델 무결성 사각지대.
- **N10. GAN 회피 ↔ OOD 탐지 군비경쟁** — 탐지회피와 분포외(OOD) 탐지를 한 프레임에. (arXiv:2506.21142) → A11 보강, B14·B1(OOD) 대응 방향.

### 📌 우선순위 (보고서 임팩트)
1. **N4 Raven + N5 SwarmFuzz** — 드론군집·은밀성 직결("1점→군집 전파", "노이즈 내 1회 주입").
2. **N1 Incalmo** — kill-chain 자동화 정량근거(37/40), Red "무한 조합" 서사 뒷받침.
3. **N8 VLM 인젝션** — "방어 AI 자체가 표적" 메타 축.

### 🔧 카탈로그 갱신 제안
- **승격(추가제안→정식):** SwarmFuzz, VLM 프롬프트 인젝션 (1차 출처 확정).
- **신규 변형 추가:** N3(멀티에이전트 레드팀), N7(센서 박탈), N9(상태기반 백도어).

## B축 — 방어 견고함 (신규) ✅

### 회복력 (Resilience / BFT / Self-healing)
- **R1. SwarmRaft ★** — Raft 합의 + P2P 거리측정 + 잔차 투표로 **GNSS 스푸핑된 위치보고를 군집이 자율 교정**. **노드 50% 손상돼도 일관성 유지**(복원오차 19m→0.28m). (arXiv:2508.00622) → B2·B5·EX1 상위호환. **가용성+**(격리/회항 아니라 *교정 후 계속 비행*). 비용 리더 O(N²), N≤17 OK.
- **R2. Reputation-PBFT(RePA)** — 평판 기반 노드 선출로 **탈취된 UAV가 합의 결정권 못 갖게**. (Springer 2025, s43926-025-00164-y) → B9 보강(내부자/노드탈취). 가용성 +/0.
- **R3. Graceful Degradation + 중복성** — 손상 UAV 철수해도 **잔존 군집이 태스크 재분배로 축소용량 임무 지속**('attrition 회복력 임계값' 설계). (SCIRP 2024; survey arXiv:2506.16786) → **가용성++**(방어=가용성 보존). 우리 핵심 원리 뒷받침.
- **R4. Quantum Adaptive Graph 자가치유** — 그래프 재구성으로 노드 손실 시 연결성 복원. ⚠️전문 미검증(개념만, Springer).

### 능동 방어 (Active Defense / MTD / Deception)
- **D1. 연합 MARL MTD ★** — FL+멀티에이전트 DRL로 IP/포트 동적 이동, **DoS 중에도 가용성 유지**. (arXiv:2506.07392) → B10 **적응형으로 격상**. 가용성 +/0(가용성을 보상에 명시).
- **D2. MTD+기만 하이브리드 + 게임이론 정량평가** — 대기행렬+진화게임으로 방어비용 대비 효과 수치화. (MDPI Computers 14(12):513 ⚠️스니펫) → B6+B10 결합 프레임. 채점식 논증 활용. 가용성 −/0.
- **D3. 선제적 ATT&CK 디코이** — 공격자가 다음 노릴 기법에 맞춰 디코이 **미리 배치**. (arXiv:2404.12783 ⚠️초록) → B6 능동화. 가용성+.

### 공격유형별 차등 복구 (우리 핵심 아이디어 직접 보강) ★★
- **C1. CPS 복구 taxonomy ★최우선** — 복구를 **공격유형·공격면별 + shallow(전용복구기 無)/deep(有)**로 체계 분류. "대다수가 CPS 복구를 안 봤다"고 지적 → **우리 '공격유형별 차등 복구'의 신규성·연구공백 뒷받침**. (arXiv:2404.04472) → 예: 스푸핑→상태추정복원(deep)/DoS→채널전환(shallow)/노드탈취→합의 재선출. 가용성+.
- **C2. RL 임무인지 자율 사고대응 ★(가용성 보상의 실사례)** — FFI(노르웨이 국방연). 보상 = **주행 중 불필요행동 −10(과잉대응 억제) + 정지 −1(가용성) + 목표 +50 + 무위협 do-nothing +1** → **보안대응 vs 가용성을 보상함수로 직접 균형** (= 우리 `(공격+방어)×가용성`을 보상으로 구현한 실사례!). (arXiv:2410.21407) → C1을 자동화·차등화하는 실행엔진. **가용성++**.

### 📌 우선순위 (보고서 임팩트)
1. **최우선:** **C1**(공격유형별 복구 taxonomy) + **C2**(가용성-보안 균형 보상) + **R1**(SwarmRaft 50% 자율교정) + **D1**(연합 DRL MTD).
2. 보강: R2(평판 PBFT), R3(graceful degradation).
3. 개념·주의(미검증): R4, D2, D3.

### 🔧 카탈로그 갱신 제안 (방어)
- B10 MTD → **적응형(연합 DRL)으로 격상**(D1) · B9 비잔틴 → R1/R2로 강화.
- **신규 정식 후보:** "**공격유형별 차등 복구(가용성 인식 RL 대응)**" (C1+C2) — 우리 ③ 보상설계의 직접 근거.

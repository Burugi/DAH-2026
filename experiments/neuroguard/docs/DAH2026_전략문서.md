## 0. 핵심 요약 (TL;DR)

- **우리 팀의 무기:** AI 에이전틱 시스템(석사 2명 + 본인) + 임베디드(UAV/UGV 도메인). → **③ AI 에이전트 아키텍처(25pt)는 만점권 목표**, 임베디드로 **①②의 도메인 현실성** 확보.
- **우리 팀의 리스크:** 순수 공격/방어 보안(침투·취약점) 전담 인력 부재 → ①(30pt)·②(25pt)에서 깊이 부족 위험. **→ 도메인 보안 속성 학습 + AI 에이전트 관점으로 프레이밍**해 정면 돌파.
- **승부처 인사이트(본선):** `점수 = (공격 + 방어) × 가용성`. **가용성이 곱셈 계수** → 공격에 성공해도 시스템을 망가뜨리면 점수 폭락. **"가용성을 지키는 외과수술식 공방"**이 우승 공식. 임베디드 팀의 실시간·가용성 감각이 강점.
- **🆕 CAGE 벤치마크 핵심 교훈:** (1) **휴리스틱+RL 하이브리드가 순수 딥RL/MARL을 이긴다**(CC3·CC4 반복 확인) → 백지+15일 팀에 최적. (2) **CC3 드론 군집 시나리오가 우리 UAV 도메인의 청사진**(공급망 트로이목마·웜 확산, Canaries/Whistles 방어). (3) **CybORG 가용성 보상 = DAH 채점식과 동일 철학**.
- **예선 전략:** 5개 항목 전부를 노린다. ③를 차별화 무기로, ①②를 UAV/UGV 도메인+AI에이전트로 엮고, ④⑤는 확실히 챙겨 **기본 20점 + ③ 25점을 안정 확보**.

## 2. 전략 기본 방향

### 2.1 우리 팀 = 이 대회의 이상적 조합

| 팀 자산 | 대회에서의 의미 |
|---------|-----------------|
| AI 석사 × 2 | ③ AI 에이전트 아키텍처(RL/LLM 에이전트), ② 방어 ML(이상탐지) |
| 임베디드 × 1 | ①② UAV/UGV 공격면·방어 현실성(MAVLink, 펌웨어, 센서, RF) |
| 본인 (임베디드 + 에이전틱) | ③ 에이전트 오케스트레이션 + 보고서 통합/PM + ① 공격 AI 프레이밍 |

> **메시지 전략:** "우리는 *AI 에이전트를 실제 무기체계(UAV/UGV) 도메인에 적용*할 수 있는 팀" — 순수 보안팀이나 순수 AI팀이 못 가지는 교집합을 강조. 이게 ④ 팀 역량에서 강력한 서사.

### 2.2 약점 보완 플랜 (보안 깊이)

- 순수 공격/방어 보안 전담이 없음 → **2가지로 메움**
  1. **도메인 보안 속성 학습 (6/25~6/30):** UAV/UGV 위협 모델, MITRE ATT&CK / ATT&CK for ICS, PX4·ArduPilot·ROS2 알려진 취약점, MAVLink/통신 링크 공격.
  2. **AI 에이전트 관점 프레이밍:** 대회의 본질은 "사람 해커"가 아니라 **AI 에이전트의 자율 공방**. 즉 우리가 잘하는 영역으로 문제를 끌어온다. → 약점이 자동 완화.
- 🆕 **검증된 우회로:** CAGE에서 **휴리스틱(규칙 기반)이 순수 RL을 이겼다**(CC3·CC4). 우리가 보안 전문 침투 기술이 약해도, **임베디드 도메인 지식 기반 휴리스틱 + RL 하이브리드**로 충분히 최상위가 가능 → 약점을 강점으로 전환.

### 2.3 "가용성" 중심 설계 철학 (본선 우승의 열쇠)

- 점수가 `(공격+방어) × 가용성`이므로:
  - **공격 에이전트:** 무차별 파괴 ✗ → **목표 달성 + 시스템 가용성 유지**하는 정밀 공격(스텔스/지속).
  - **방어 에이전트:** 위협 차단 시 **서비스 다운 최소화**(격리·페일오버·점진 대응). 과잉 차단으로 SLA 깨면 자해.
  - 임베디드의 실시간·가용성 감각 = 이 철학을 자연스럽게 구현. 보고서 ②③에서 "가용성 인식형(availability-aware) 에이전트"로 명시.
- 🆕 **선행 근거:** CybORG(CAGE CC4) 보상은 red 행동이 아니라 **green(정상 사용자) 작업 실패**를 페널티로 삼음 = `×가용성`과 동일한 사고. → 보고서에서 "검증된 설계 철학"으로 인용 가능.

## 3. 예선 전략 (D-15 스프린트)

### 3.1 항목별 공략

#### ① 공격 시나리오 설계 (30pt) — *창의성 + 기술적 완성도*
- **구성:** 구체적인 UAV/UGV **킬체인** 1~2개를 MITRE ATT&CK 단계로 매핑.
  - 예) `GCS(지상통제) 침투 → MAVLink 명령 주입/위조 → GPS 스푸핑 → 항법 교란/탈취 → 임무 거부(DoS)` 를 **AI 공격 에이전트가 상황 적응적으로** 수행.
- **🆕 벤치마크 차용(CC3 드론 시나리오 = 우리 도메인 청사진):** **펌웨어 하드웨어 트로이목마(공급망 공격) → 휴면 → 활성화 → 웜이 인접 드론(MANET)으로 확산 → 정보 탈취/허위정보 주입**. CC3의 검증된 위협 모델을 우리 UAV 킬체인 템플릿으로 활용(`CAGE_Challenge_리서치.md` §2.3).
- **창의성 포인트:** 정적 익스플로잇이 아니라 **AI 에이전트가 정찰→판단→다단계 공격을 자율 전개**(적응형, 회피형). 위성망 지연·단절을 악용하는 시나리오.
- **완성도 포인트:** 대상 시스템/프로토콜/취약점/페이로드를 구체적으로. 가정과 전제, 성공 조건 명시. 행동을 MITRE ATT&CK 기법 ID(예: Exploit→T1210, Impact→T1489)에 매핑하면 전문성↑.

#### ② 방어 전략 수립 (25pt) — *견고성 + 실현 가능성*
- **계층 방어 아키텍처:**
  - 탐지: 텔레메트리 이상탐지(시계열), 네트워크 IDS, 행위 기반 탐지, **GPS 스푸핑 탐지(센서 퓨전 교차검증)**, MAVLink 인증/무결성.
  - 대응: **가용성 보존형 자율 대응**(의심 노드 격리, 페일세이프 항법, 점진적 차단, 디코이).
- **🆕 벤치마크 차용(검증된 우승 방어 패턴):**
  - **Canaries + Whistles**(CC3 우승작): 정상 드론이 ID **하트비트** 브로드캐스트 → 멈추면 감염 추정(Canary), 감염 ID를 군집 전체로 **재전파**(Whistle). 드론 군집 방어에 즉시 적용 가능.
  - **기만(Deception)**: CC2 우승 레시피 = **디코이/허니팟의 탐욕적 배치**. 디코이를 핵심 방어 레이어로.
- **실현 가능성 포인트:** 클라우드 위성망 모사 환경에서 실제 구현 가능한 컴포넌트로 기술. 오픈소스(허용 라이선스) 기반 스택 제시.

#### ③ AI 에이전트 아키텍처 (25pt) — ***우리의 차별화 무기***
- **에이전트 구성:** 공격 에이전트 / 방어 에이전트 / **정상 사용자(Green) 모델**(가용성 평가 기반) (+ 필요시 조정자/플래너). → CAGE의 **Blue/Red/Green 3-에이전트 정식화**(검증된 틀)를 출발점으로.
- **설계 상세(구체성):**
  - **관측공간(Observation):** 텔레메트리, 네트워크 플로우, 시스템 로그, 알람.
  - **행동공간(Action):** (공격) 정찰/침투/주입/지속, (방어) 분석/격리/복구/디코이/**트래픽 차단**.
  - **의사결정 코어 — 🆕 하이브리드 권장:** **도메인 휴리스틱 + RL(PPO)** 하이브리드 + 상위 **LLM 에이전틱 추론(계획·도구사용)**. ※ CAGE에서 순수 딥RL/MARL보다 휴리스틱·하이브리드가 우월했음 → **혁신성 + 실현가능성 동시 확보**.
  - **학습:** 자가대전(self-play)·커리큘럼 학습. RL은 **PPO(Stable-Baselines3)** 표준, **DQN은 부분관측·노이즈에서 불안정하니 회피**. 무작위 토폴로지 대비 **GNN/그래프 표현**(Cybermonic GNN+PPO: 네트워크 크기 자동 확장) 검토.
- **혁신성 포인트:** ① LLM 에이전트(novel 상황 추론) + RL/휴리스틱(빠른 제어) 결합, ② **가용성 인식 보상함수**, ③ 임베디드 제약(실시간/자원)을 반영한 sim-to-real 고려.
- **벤치마크 근거:** CybORG/CAGE의 관측/행동/보상 정식화 + MITRE ATT&CK 매핑을 참조해 **검증된 틀 위에 우리 혁신을 얹는다** (상세 → `CAGE_Challenge_리서치.md` §3·§5).

#### ④ 팀 역량 소개 (10pt)
- 멤버별 전문성 → 대회 기둥에 1:1 매핑(표). AI석사=③②, 임베디드=①②도메인, 본인=③오케스트레이션+PM.
- "**임베디드 × AI 에이전트**" 교집합을 팀의 정체성으로 서술.

#### ⑤ 문서 완성도 (10pt, 자동 평가)
- 일관된 구조·다이어그램·표·페이지 번호·목차·용어 통일. 오탈자 0. **마감 1일 전 완성 후 교정.** (자동 평가이므로 형식만 지켜도 만점권.)

## 5. 벤치마크 — CAGE Challenge에서 가져올 것 ✅검증완료

> CAGE Challenge = **CybORG**(Cyber Operations Research Gym) 기반 **자율 사이버 방어 RL 벤치마크 시리즈**(TTCP 5개국 / 호주 DSTG 주관, 2021~2024 CC1~CC4). DAH와 가장 가까운 선행 사례.
> 📂 **전체 상세 리서치: `CAGE_Challenge_리서치.md`** (출처·정식화·각 대회·우승 기법 전부 정리)

### 5.1 우리에게 결정적인 교훈 5가지 (actionable)

1. **🔑 휴리스틱+RL 하이브리드가 순수 RL을 이긴다** — CC3·CC4에서 **잘 설계된 규칙 기반(휴리스틱) 에이전트가 딥RL/MARL을 압도**(CC3 우승작 −1577 vs PPO −7617, 약 5배). **백지+15일 우리 팀에 최적의 메시지**: 순수 딥RL에 올인하지 말고, **임베디드 도메인 지식 기반 휴리스틱 + (선택적) PPO 하이브리드**로 간다. → ③에 "하이브리드 아키텍처"로 명시하면 **혁신성 + 실현가능성** 동시 확보.
2. **🚁 CC3 드론 시나리오 = 우리 UAV 도메인의 청사진** — MANET 드론 군집, **펌웨어 하드웨어 트로이목마(공급망)**, **웜 확산**이 위협 모델. → ① 공격 킬체인 템플릿으로 직접 차용. 방어는 우승작의 **Canaries(드론별 하트비트→멈추면 감염 추정) + Whistles(감염 ID 군집 전파)** 패턴을 ②에 구체적·구현가능한 방어로 인용.
3. **🎭 기만(Deception)이 1급 방어 행동** — CC2 우승 레시피 = **PPO + 탐욕적 디코이 배치**. → ②에 디코이/허니팟/canary를 **핵심 방어 레이어**로 설계.
4. **📈 "가용성 = 정상 사용자(green) 영향" 보상 = DAH 채점식과 동일 철학** — CybORG CC4 보상은 red 행동이 아니라 **green 작업 실패**를 페널티로 삼음. DAH의 `(공격+방어)×가용성`과 **정확히 같은 사고** → 우리 "가용성 인식 에이전트" 논리의 **강력한 선행 근거로 보고서에 인용**.
5. **🛠 알고리즘보다 관측·보상 엔지니어링** — PPO(Stable-Baselines3) 표준, DQN은 부분관측·노이즈에서 불안정해 회피. 무작위 토폴로지엔 **GNN/그래프 표현**(Cybermonic GNN+PPO, 네트워크 크기 자동 확장) 고려. MARL은 무작위 초기화 적응 실패·파라미터 공유 병목으로 자주 실패 → **본선에서 섣부른 MARL보다 단일 에이전트 DRL+휴리스틱이 안전**.

### 5.2 ③ 아키텍처 정식화 차용 (검증된 틀)

- **3-에이전트 모델:** Blue(방어)/Red(공격)/**Green(정상 사용자=가용성·노이즈)**. Green을 명시적으로 모델링하는 게 핵심(가용성 평가의 기반).
- **POMDP 정식화:** 관측(텔레메트리·네트워크 플로우·로그·알람) → 행동(Blue: Monitor/Analyse/Remove/Restore/Decoy/**BlockTraffic**, Red: 정찰→발견→익스플로잇→권한상승→Impact) → 보상(음수 페널티 누적, 가용성 중심).
- **행동의 MITRE ATT&CK 매핑** (예: Exploit→T1210, Impact→T1489) → 보고서 ①의 전문성·완성도↑.
- 이 검증된 틀 **위에** 우리 혁신: (a) **LLM 에이전틱 추론**(novel 상황·계획·도구사용) + RL/휴리스틱(빠른 제어), (b) **가용성 인식 보상함수**, (c) **임베디드 sim-to-real**.

### 5.3 차별화 무기 — sim-to-real (우리 임베디드 강점)

- CAGE의 **공인된 한계** = "시뮬레이션 과적합 · 실제망 준비도 의문(*Beyond games?*)". 스크립트형 red·추상 관측 때문에 "게임"에 가깝다고 운영진도 인정.
- → 우리는 **임베디드 역량으로 실제 UAV/UGV 시스템 제약(실시간성·자원·펌웨어·센서)을 반영한 현실성**을 보고서에서 부각. 심사위원에게 **"이 팀은 시뮬레이션을 넘어 실전을 안다"**는 강한 인상 → ①②③ 전반의 가산점.

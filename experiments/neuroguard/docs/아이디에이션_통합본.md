# ① 공격 시나리오 (순위별)

## 🥇 1위 — A1. 공급망 펌웨어 트로이목마 → 군집 웜 확산 `[sim✅]` ★대표
- **근거:** 공급망(부품·펌웨어·OTA) 신뢰 검증이 취약 + **CAGE CC3 공인 위협모델** + **IoT Goes Nuclear(ZigBee 연쇄반응, IEEE S&P 2017)** 무선 웜 도시규모 확산 실증 + **Maldrone(2015)** 드론 백도어 PoC.
- **예상 성능:** 1대 감염→군집 전체 확산(1→N). 잠복형이라 **가용성 유지하며 정보 유출** → 고득점형. CC3 `RedDroneWorm`으로 직접 실현(`red_owned`·`reward` 관측).
- **검토 의견:** 현실성+벤치마크+sim 삼박자로 **대표 시나리오 확정.** 단 "자가전파 웜 야전 확산"은 실사건보다 연구·시나리오 → 보고서엔 *"검증된 위협모델"*로 프레이밍.

## 🥈 2위 — A2. GCS 침투 → MAVLink 명령 주입 `[개념]`
- **근거:** MAVLink 1.0 평문·무인증 + **CVE-2026-1579(PX4, CVSS 9.8)+CISA** 실증(SERIAL_CONTROL→셸).
- **예상 성능:** 합법 채널로 직접 조종 → 기체 탈취/임무 무력화. behavior blending으로 저강도 지속 가능.
- **검토 의견:** **CVE로 근거 최강.** sim 미구현([개념])·군용 암호화 링크엔 제한 → *"Wi-Fi/평문 링크 한정"* 단서 필요. MAVLink 2.0도 서명 기본 OFF·암호화 아님.

## 🥉 3위 — A3. GPS 스푸핑 → 항법 기만 `[sim✅]`
- **근거:** 민간 GNSS 비암호화 + **Humphreys(ION GNSS 2012)** DHS 앞 드론 GPS 탈취 첫 실증. 점진 드리프트로 INS 잔차 회피.
- **예상 성능:** 기체는 계속 비행(가용성 유지)하며 위치·임무만 왜곡 → 공격+가용성 동시. CC3 `gps_spoof`(pos_rep drift)로 실현.
- **검토 의견:** 교과서적·sim 실현. 군용 M-code엔 불가(민간 한정 명시). B2(CUSUM) 방어와 짝 시나리오로 강력.

## 4위 — A7. RF 재밍·미코닝 `[sim✅]`
- **근거:** 전자전 최흔 위협 + **OPSGROUP 2024**(항공 GPS 교란 일 1,500편). 미코닝=진짜 신호 재전송.
- **예상 성능:** 통신 두절 → **가용성 직접 타격**. CC3 `snr -= drop`로 실현.
- **검토 의견:** 탐지는 쉬우나(SNR↓) "끊긴 채 임무 유지"가 방어 난제 → 가용성 서사에 핵심. 재밍≠미코닝 정의 분리.

## 5위 — N5. SwarmFuzz (1대 스푸핑→군집 연쇄) `[개념]` 🆕
- **근거:** **SwarmFuzz(DSN 2023)** — 1대 GPS 스푸핑이 군집 동역학 타고 연쇄 충돌(무작위 대비 10배).
- **예상 성능:** 표적 1점 공격의 비대칭 파급 → 군집 전체 마비. A3·A14 전파 메커니즘.
- **검토 의견:** "1점→군집" 메시지가 강력. B5(확산차단)의 존재이유. 출처 확정 → 정식 항목 승격.

## 6위 — N4. Raven (은밀 의미론 공격) `[개념]` 🆕
- **근거:** **Raven(USENIX Sec 2025)** — 센서 노이즈 한도 내 **1회 주입**으로 군집 교착/몰이/충돌(탐지 회피).
- **예상 성능:** 임계 기반 탐지 무력화 → 은밀성 극대. A6/A10 격상.
- **검토 의견:** "왜 임계 탐지만으론 부족한가"의 직접 근거. 단 USENIX PDF 스니펫 검증 → 정량 인용 시 원문 확인.

## 7위 — N1. Incalmo (LLM kill-chain 자동) `[개념]` 🆕
- **근거:** **Incalmo(2025)** — LLM이 선언적 태스크 + attack-graph로 다중호스트 kill-chain 자율 실행, **37/40 성공**.
- **예상 성능:** "정해진 행동, 무한한 조합"을 자동 연쇄 → 창의적 루트 생성.
- **검토 의견:** 우리 Red 행동집합·자율성의 학술 근거. 원문 확인됨(강력).

## 8위 — N8. VLM/VLA 프롬프트 인젝션 (메타 공격) `[개념]` 🆕
- **근거:** 이미지 속 스테가노 은닉 명령으로 시각-언어 모듈 정렬 붕괴(SOTA VLM 4종 취약).
- **예상 성능:** ISR 영상·표적식별 AI를 통째 탈취 → 인지·행동 장악.
- **검토 의견:** "방어 AI를 쓰면 방어 AI가 표적" 메타 논점. 우리 LLM 하이브리드(C12) 자기위협 분석.

# ② 방어 전략 (순위별)

## 🥇 1위 — 가용성 보존형 자율 대응 (B7) ★채점 직결
- **근거:** `×가용성` 채점의 정공법 + **CAGE Restore −1 페널티**(운영중단 비용). 대응 사다리: 모니터링→격리→페일세이프→복원.
- **예상 성능:** 위협 심각도 vs 가용성 손실을 저울질해 **최소침습** 선택 → 본선 점수 직결.
- **검토 의견:** 모든 방어의 상위 원칙. ③ 보상설계와 직결. 가장 중요한 방어 서사.

## 🥈 2위 — 공격유형별 차등 복구 (가용성 인식 RL) ★★ 🆕 우리 핵심
- **근거:** **C1 CPS 복구 taxonomy(arXiv:2404.04472)** 공격유형별 shallow/deep + "연구공백" + **C2 RL 사고대응(FFI, arXiv:2410.21407)** 보상 `과잉대응−10/정지−1/임무+50` = 우리 채점식 구현.
- **예상 성능:** 스푸핑→상태추정 복원 / DoS→채널 전환 / 노드탈취→합의 재선출 등 **공격에 맞는 복구** → 가용성 ++.
- **검토 의견:** **우리 신규성+근거 동시 확보**(연구공백 지적). ③ 보상설계의 결정적 인용. 최우선.

## 🥉 3위 — Canary & Whistle (B5) `A1 대응`
- **근거:** **CC3 우승작(휴리스틱>PPO 5배, arXiv:2312.04940)**. 하트비트 멈춤=감염→격리·전파.
- **예상 성능:** 감염 노드만 격리(가용성 유지)하며 확산 차단. 고득점·저구현비용.
- **검토 의견:** A1 웜의 직접 대응. snr 교차검증으로 "감염 vs 재밍" 구분 시 오탐↓.

## 4위 — GPS 스푸핑 탐지 (B2) `A3 대응` `[sim✅]`
- **근거:** INS-GNSS 잔차·RAIM(Khanafseh PLANS'14) + **CUSUM**(느린 드리프트 누적 탐지). CC3 multisensor.
- **예상 성능:** 고정 임계의 약점(드리프트 회피)을 누적탐지로 보완 → 견고성 입증.
- **검토 의견:** "고정임계 vs CUSUM" 실험이 ②견고성 근거(그래프거리). 스푸핑 의심 시 INS 페일세이프로 가용성 유지.

## 5위 — SwarmRaft 합의 자율교정 (R1, B9 강화) 🆕
- **근거:** **SwarmRaft(arXiv:2508.00622)** — Raft 합의+거리측정으로 **50% 손상돼도 위치 자율 교정**.
- **예상 성능:** 격리/회항이 아니라 *교정 후 계속 비행* → **가용성 +**. 복원오차 19m→0.28m.
- **검토 의견:** 회복력 축의 강력한 신규. 비용 리더 O(N²)(N≤17 OK). R2(평판 PBFT)와 함께 비잔틴 방어.

## 6위 — 기만/디코이 (B6)
- **근거:** **CC2 우승 레시피(PPO+탐욕적 디코이)** + 허니드론 하이퍼게임 DRL(ACM ToIT'24).
- **예상 성능:** 공격자 조기 노출+시간 낭비. 디코이는 실자산 분리 → 가용성 무손실.
- **검토 의견:** "역공" 대체 능동방어. D3(선제 ATT&CK 디코이)로 고도화 가능.

# ③ AI 에이전트 아키텍처 (순위: 핵심 + 변형)

## 🥇 핵심 설계 (채택)
**Blue/Red/Green 3-에이전트 + 휴리스틱+PPO+LLM 하이브리드 + 가용성 인식 보상 + 안전가드**
- **근거:** CAGE CC4 표준(Blue/Red/Green) + **"휴리스틱이 모든 MARL 능가"(CC3·CC4)** → 휴리스틱 baseline+PPO 보강이 검증된 우승 공식. LLM은 장기 누적보상서 RL에 밀려(arXiv:2505.04843) **오프라인**(보상설계·계획·설명)으로.
- **예상 성능:** 15일·백지 팀에 현실적이면서 견고. action-masking으로 RL 탐색↓, 안전가드로 가용성·안전 위반 차단. CAGE 휴리스틱 우승 사례가 성능 하한 보장.
- **검토 의견:** **혁신성+실현가능성 동시.** R_red는 가중합(곱셈✗). 보상=`방어이득−침해손실−λ·가용성손실`(C2가 실사례). 환경은 CC4/CybORG++ 채택(직접구현 ✗).

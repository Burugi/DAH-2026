# PART 1. 무인체계 신호·코드가 어떻게 도는가 (기초)

## 1.1 핵심 그림 — 제어/추정 루프
드론·UGV의 심장은 **"센서 → 상태추정 → 제어 → 구동"** 루프다.

```
[센서]                 [상태추정]            [제어]          [구동]
IMU(자이로/가속도) ─┐
GPS/GNSS ───────────┼─→ EKF/칼만필터 ─→ 위치·자세 추정 ─→ 제어기(PID) ─→ 모터/조향
기압계·지자기 ──────┘     (센서 융합)         (state)       (목표 vs 현재)
        ↑                                                         │
        └──────────────── 물리 세계(움직임) ←──────────────────────┘
              텔레메트리(MAVLink) ↕ GCS(지상통제)
```
- **FCU 루프**: 수백 Hz(실시간). **텔레메트리**: 1~10 Hz로 GCS에 상태 보고 + 명령 수신.
- 🔑 **공격 표적은 대부분 "상태추정"**: GPS/센서를 속이면 → 추정이 틀어지고 → 제어가 틀어지고 → 물리적으로 엉뚱하게 움직인다.

## 1.2 MAVLink — 명령·텔레메트리가 흐르는 길
GCS ↔ FCU 메시지 프로토콜. 주요 메시지:

| 메시지 | 방향 | 내용 | 주기 |
|--------|------|------|------|
| HEARTBEAT | 양방향 | "살아있음" + 비행모드 | 1Hz |
| GLOBAL_POSITION_INT | 드론→GCS | 위치/고도/속도 | 수 Hz |
| ATTITUDE | 드론→GCS | 자세(roll/pitch/yaw) | 수 Hz |
| GPS_RAW_INT | 드론→GCS | GPS 원시(위성수·정확도) | 수 Hz |
| COMMAND_LONG | GCS→드론 | 명령(MAV_CMD_*: 이륙·RTL·무장) | 이벤트 |
| MISSION_ITEM | GCS→드론 | 항로점 | 이벤트 |

- **보안:** MAVLink **v1 = 평문·무인증** → 누구나 스니핑/위조 가능. v2 = **메시지 서명(HMAC-SHA256)** 지원하나 미적용 사례 多.
- 🔑 A2(명령 주입)·A8(C2 탈취)·A4(재전송)의 무대. 방어 B3 = v2 서명 + 시퀀스/논스로 재전송 거부.

## 1.3 GNSS/GPS — 위치는 어떻게 정해지나
- 위성 ≥4개의 **의사거리(pseudorange)** 측정 → 삼변측량 → (x,y,z,시각). 민간 L1 C/A는 **비암호화**.
- **스푸핑 메커니즘:** 공격자가 **더 강한 가짜 위성신호**를 송신 → 수신기가 가짜에 락온 → 보고 위치가 가짜로 이동. **점진 드리프트**(천천히 끌기)면 운용자·필터가 눈치 못 챈다.
- **EKF 융합:** GPS(절대위치) + INS(관성, 상대변화)를 합친다. 정상이면 둘이 일치, **스푸핑이면 GPS와 INS가 어긋남(잔차↑)** → 이게 탐지 단서(B2).

## 1.4 통신 — SNR·링크·재밍
- **SNR**(신호대잡음비, dB): 높을수록 통신 양호. **링크 성립 = SNR > 임계**.
- **재밍(jamming):** 같은 대역에 잡음을 쏘아 **SNR을 떨어뜨림** → 링크 두절. **미코닝(meaconing):** 진짜 신호를 녹화 후 재방송해 기만.
- **위성망(SATCOM):** 고지연 + 간헐 단절 → 단절 구간이 공격 기회(A4), 방어는 단절에도 자율 유지(가용성).

## 1.5 ROS2/DDS — 자율 스택의 메시지 버스
- 컴패니언 컴퓨터(자율주행·영상)의 노드들이 **topic**으로 pub/sub. 예: `/scan`(LiDAR), `/cmd_vel`(속도명령), `/odom`(주행거리계).
- **DDS**가 노드 자동 디스커버리·전송 담당. **SROS2(보안) 미적용 시 인증 없음** → 악성 노드가 합류해 `/scan`에 유령 장애물, `/cmd_vel`에 위조 명령 주입(A5).

## 1.6 🎯 팀 레포(CC3 lab)에선 이 신호들이 이렇게 모델링된다
> 핵심: CybORG는 **네트워크(웜)**만 진짜로 시뮬, **위치/GPS/통신은 `fleet.py`가 합성 신호 + ground-truth 라벨**로 생성(2채널 구조).

| 실제 개념 | 레포 변수 (`fleet.py`/`defense.py`) | 정의 |
|-----------|-----------------|------|
| 실제 위치 | `pos_true[t,e]` | 함대 진짜 좌표 |
| 보고 위치 | `pos_rep` | GPS가 보고하는 좌표(스푸핑 시 왜곡) |
| GPS 오차 | `gps_err = ‖pos_rep − pos_true‖` | 클수록 항법 기만 |
| 통신 품질 | `snr` (≈20dB + 잡음) | 재밍 시 하락 |
| 링크 | `link_up = snr > 6` | 통신 가능 여부 |
| GPS 스푸핑 | `pos_rep[t] += drift·(k+1)` | **누적 드리프트** |
| 재밍 | `snr[targets] −= drop(≈22)` | SNR 급락 |
| 웜 감염 | CybORG sessions → `red_owned[t,e]` | 1 = 감염 |
| 시뮬 보상 | `reward[t]` | CybORG blue 보상(음수) |
| 재밍 탐지 | `det_jam = snr < snr_thresh(6)` | threshold 검출기 |
| GPS 탐지 | `det_gps = ‖pos_rep − IMU‖ > gps_thresh(8)` | multisensor(IMU 교차검증) |
| 복구 대응 | `safe_mode`: 위치를 IMU 추정으로 보정 | 스푸핑 위치 교정 |
| 격리 대응 | `isolate` (det_jam 또는 det_gps) | 의심 엔티티 격리 |
| 채점(현재) | 탐지 P/R/**F1** + `gps_err_before→after` | 라벨 대비 |

🔑 **여기서 바로 구체화 포인트가 보인다:** multisensor 탐지는 `‖pos_rep − IMU‖ > 8`이라는 **고정 임계** → 공격이 drift를 8 밑으로 천천히 끌면 **탐지 회피**. (Part 2에서 해결)

# PART 2. 시나리오 구체화 (신호 · 흐름 · 의사로직)

> sim에서 실제로 도는 신호로 각 아이디어를 구체화. 의사로직은 설명용(레포 수정 아님).

## 2.1 A3 GPS 스푸핑 ↔ B2 탐지 — 가장 교육적

**① 공격(A3) 신호 수준**
- 목표: `pos_rep`를 천천히 끌어 항법 기만하되 **IMU 잔차 < 임계** 유지(탐지 회피).
- 레포 매핑: `gps_spoof`의 `drift`를 키우면 `gps_err`↑지만 `det_gps`에 걸린다. **작게 점진**이 은밀.
```text
# 공격 의사로직 — 은밀 드리프트
for t in spoof_window:
    rate = min(target_rate, (gps_thresh - margin) / dt)   # 탐지임계 밑으로 유지
    pos_rep[t] = pos_rep[t-1] + rate                       # 누적 이동
```

**② 방어(B2) 신호 수준 — 현재 레포 multisensor**
```text
imu      = pos_true + cumulative_noise          # INS 추정(작은 누적 노이즈)
residual = ‖pos_rep − imu‖
if residual > gps_thresh(8): det_gps = True → safe_mode(위치를 imu로 교정)
```

**③ 🆕 구체화된 개선(보고서용)**
- 고정 임계의 약점(느린 드리프트 회피) → **누적합 변화탐지(CUSUM)** 또는 **적응 임계**: 잔차의 *추세*를 보면 느린 드리프트도 검출된다.
```text
S = max(0, S + (residual − k))     # CUSUM 누적
if S > h: 스푸핑 의심                # 느린 드리프트도 누적되어 결국 검출
```
→ **"고정 임계 vs CUSUM" 비교 실험**이 그대로 **②방어의 견고성 근거**가 된다(보고서 그래프거리).

## 2.2 A7 재밍 ↔ 방어 — 신호 두절과 가용성
**공격:** `snr[targets] −= drop` → `link_up = 0`. 부분 재밍(일부 드론)·전체·반응형 변형.
**방어:** `det_jam = snr < 6`(threshold). 단 **탐지는 쉬워도 "두절 자체"가 가용성 손실** → 방어의 본질은 *재밍 중에도 임무 유지*.
- **B10 주파수 도약 / B12 RF 지문**(개념) + **자율 페일세이프**(링크 끊겨도 INS로 임무 지속).
- 🔑 **구체화 인사이트(중요): 재밍 vs 감염 구분** — 통신이 끊겼을 때 **재밍이면 `snr↓`, 웜 감염이면 `snr` 정상**. 두 신호 교차검증으로 오탐을 막는다(2.3과 직결).

## 2.3 A1 웜 ↔ B5 Canary & Whistle — 군집 확산과 격리

**① 공격(A1):** CybORG에서 red가 이웃 드론을 exploit → session 획득 → 확산. 관측 = `red_owned`↑, `reward`↓.
- 레포 red: `RedDroneWorm`(확산형) / `DroneRedAgent`(FSM). `starting_num_red`·`red_spawn_rate`로 강도 조절.

**② 방어(B5) 의사로직**
```text
# Canary: 각 드론이 주기적으로 하트비트 송신
every k steps: broadcast heartbeat(id, seq)

# 탐지: 하트비트 누락 → 감염 의심 (단, 재밍 아닌지 snr로 교차검증!)
if missed_beats(j) > m and snr[j] is normal:   # snr 정상인데 침묵 = 감염 의심
    suspect(j) = True
    broadcast Whistle(j)                        # 군집 전체에 전파

# 대응: 이웃이 의심 노드 격리/불신 (가용성 보존: 격리만, 임무는 지속)
on Whistle(j): neighbors.isolate_link(j)
```
- 레포 blue policy `react`(감염된 자기 드론 세션 제거)와 결합 가능. **CC3 우승작이 바로 이 휴리스틱.**
- 🆕 **구체화:** "감염 vs 재밍" 구분(snr 교차검증)이 핵심 — **combined 시나리오(웜+재밍 동시)**에서 오탐/미탐을 줄이는 차별점.

## 2.4 B7 가용성 보존 대응 — "얼마나 세게 대응할까"의 비용 계산

대응 사다리 + 가용성 비용:

| 단계 | 행동 | 가용성 영향 | 레포 매핑 |
|------|------|-------------|-----------|
| 1 | 모니터링 강화 | 0 | (관측) |
| 2 | 의심 링크 격리 | 작음(1대 이탈, 임무 지속) | `isolate` |
| 3 | 페일세이프(INS·안전모드) | 중간(기능 제한) | `safe_mode` |
| 4 | 복구/재시작 | **큼(다운타임)** | `retake`/restore |

```text
# 의사로직 — 최소침습 선택
cost_isolate = 1 드론 이탈
cost_restore = 다운타임 × 임무가중
action = argmin(잔존위협 + 가용성손실)   # 위협 작으면 격리, 크면 복구
```
🔑 `(공격+방어)×가용성`에서 **과잉대응이 곧 감점** → 이 사다리가 ②③의 핵심 논리.

## 2.5 C 보상함수 — sim 신호로 구체화

추상 보상을 **레포 신호로** 내려쓰면:
```text
avail        = mean(link_up) × (미격리·정상가동 비율)        # 0..1 (가용성)
침해손실      = Σ red_owned                                  # 감염 드론 수
탐지이득      = F1(det, label)  또는  조기탐지 보너스
R_blue(step) = w1·탐지이득 − w2·침해손실 − λ·(1 − avail)     # 덧셈·희소
대회근사 score = (공격점수 + 방어점수) × avail               # 채점식 그대로
```
→ "보상을 sim 신호에 매핑"한 이 표가 ③의 **구체성** 점수에 직결. (R_red는 검증대로 **가중합**)

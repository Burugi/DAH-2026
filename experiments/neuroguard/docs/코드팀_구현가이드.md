# DAH 2026 — 코드팀 구현 가이드 (핸드오프)

> **목적:** 논문형식 보고서의 실험을 위한 **시나리오 + 구현 방법**. 팀 레포 `Burugi/DAH-2026`(CybORG CC3 기반) 확장.
> **합의된 방향(코드팀):** ① 벤치마크 = **CybORG 표준**(LIG 채점식 미채택, 환경 대수정 회피) ② Action **소수 제한**(CybORG 기존 집합, 자체 20개 발명 ✗) ③ 모델 = **규칙기반 + RL 하이브리드** ④ 결론 = 정해진 action 내 모델 비교 → 최고 모델 본선.

---

## 0. 핵심 원칙 (먼저)
1. **CybORG를 거의 수정하지 않는다.** 보상·시각화·평가 다 기존 것 재사용 → CAGE 공식 논문을 레퍼런스로.
2. **Action space는 CybORG가 주는 걸 그대로.** 우리는 *행동을 발명*하는 게 아니라 *그 행동들을 잘 고르는 정책(모델)*을 비교한다.
3. **범용성:** 본선 action이 다를 수 있으니, 모델 입출력을 **의미 단위(관측→행동 카테고리)로 추상화**해 재매핑 가능하게.

---

## 1. Action Space 정의 (논문 §Method 표로)

팀 레포는 **CC3(DroneSwarm)** 기반. CybORG가 주는 blue 행동을 그대로 채택:

| 행동(의미) | CybORG/레포 매핑 | 비고 |
|-----------|------------------|------|
| 대기 | `Sleep` | 무행동 |
| 세션 제거 | `RemoveOtherSessions` | 감염 세션 정리(레포 remove/react) |
| 제어 탈환 | `RetakeControl` | 감염 드론 회수(레포 retake) |
| (확장) 격리 | `BlockTraffic`(있으면) | 존/링크 차단 |
| (확장) 기만 | `DeployDecoy`(CC4식) | 디코이 |

- **현재 레포 blue 정책 5종**(sleep/random/remove/retake/react)은 이미 위 행동을 래핑 → **그대로 baseline.**
- **행동 수:** CC3는 소수(≈3~4 코어). CAGE CC4는 8개. → **CC3 유지 권장**(레포 구축됨). 더 풍부히 원하면 CC4 환경으로 이전(작업량↑).
- 📌 *자체 20개 발명 금지* — CybORG 행동집합을 §Method에 **표로 명시**하면 끝. 본선 대비 범용성은 "행동을 탐지/제거/탈환/격리/기만/대기 카테고리로 추상화"로 서술.

---

## 2. 비교할 모델 (논문의 핵심 실험)

정해진 action 내에서 **성능 비교**:

| 모델 | 정의 | 구현 위치 | 우선 |
|------|------|-----------|:--:|
| **M1. 휴리스틱 baseline** | 규칙(react/remove/retake) + **Canary/Whistle** 추가 | `agents.py` `blue_action()`에 분기 추가 | ★ 먼저 |
| **M2. PPO(RL)** | Stable-Baselines3 PPO | PettingZoo 래퍼 위 학습 스크립트 신규 | ★ |
| **M3. 하이브리드** | 휴리스틱=action-mask + PPO | M1을 마스킹 함수로, M2가 나머지 결정 | ★★ 메인 |
| M4. GNN+PPO | 네트워크 그래프 표현 | PyTorch Geometric(여유 시) | 옵션(범용성) |
| M5. 앙상블 | 다수 정책 투표 | (여유 시) | 옵션 |

> CAGE 교훈: **M1(휴리스틱)이 의외로 강함 → M1을 제대로 만들고 M3로 보강**이 현실적 우승 루트.

## 3. 구현할 시나리오 (= sim 실현 가능한 것만)

CybORG/레포에서 *실제로 도는* 공격 = **웜·재밍·GPS 스푸핑**:

| ID | 시나리오 | 레포 매핑 | config |
|----|----------|-----------|--------|
| S1 | **웜 확산**(A1) | `red_class: RedDroneWorm` (실제 sim) | baseline/defended |
| S2 | **GPS 스푸핑**(A3) | `attacks: gps_spoof` (fleet.py 합성) | defended |
| S3 | **재밍**(A7) | `attacks: jam` (fleet.py 합성) | fsm_red |
| S4 | **복합**(웜+재밍+GPS) | combined config | combined |
| (red 변형) | FSM red | `red_class: DroneRedAgent` | fsm_red |

> A2/A4/A5/A6(MAVLink·SATCOM·ROS2·센서)는 **CybORG 미모델 → 논문 §위협모델에서 *개념적 위협*으로만 서술**(구현 X). 실험은 S1~S4.

## 4. 구현 방법 (레포 기준 단계별)

### 4-0. 환경 (README대로)
```
python -m venv venv && venv\Scripts\activate
git clone https://github.com/cage-challenge/CybORG && pip install -e ./CybORG --no-deps
pip install -r requirements.txt
# (M2 추가) pip install torch stable-baselines3
```

### 4-1. M1 휴리스틱 (Canary/Whistle 추가) — `agents.py`
- `blue_action()`에 새 정책 `"canary"` 분기 추가:
  - 각 드론 하트비트 추적 → 누락 + `snr` 정상이면 감염 의심 → `RetakeControl`/`RemoveOtherSessions`.
  - "감염 vs 재밍" 구분: `snr` 정상=감염 / `snr↓`=재밍.
- 의사로직은 `구체화_신호메커니즘.md` §2.3 참고.

### 4-2. M2 PPO — 신규 `train_ppo.py`
- `run.py`의 `PettingZooParallelWrapper(env=cyborg)` 재사용.
- 관측 평탄화(`FixedFlatWrapper`/`BlueTableWrapper` 류), 행동=정수 인덱스.
- SB3 `PPO("MlpPolicy", env)` 학습 → 정책 저장.
- **보상 = CybORG 기본**(수정 X).

### 4-3. M3 하이브리드 — action masking
- M1 규칙으로 **명백한 상황은 즉시 처리**(위험·불가 행동 마스킹) → 나머지를 M2 PPO가 결정.
- (간단 구현) PPO 행동 위에 휴리스틱 override 레이어.

### 4-4. 평가·시각화 (전부 재사용!)
- `run.py` → `results/<run_id>/` (log.csv·arrays.npz·meta.json[F1]).
- `analyze.py` → 전 run 비교 `summary.csv`.
- `plot.py <run_id>` → reward/compromise/snr/gps/defense 그래프.
- `render.py <run_id>` → pygame 시각화.
- **벤치마크 지표:** CybORG `reward`(누적), `red_owned`(감염 수), 방어 `F1`(jam/gps), `gps_err_before→after`.

## 6. 범용성 & 리스크 (논문 §논의)
- **본선 action 다를 수 있음** → 모델을 **의미 단위 관측/행동**으로 설계해 재매핑(특히 M4 GNN은 크기 무관 → 범용성↑).
- **휴리스틱이 강함**(CAGE) → M1 강화 + M3 하이브리드 안전망.
- **CybORG 보상 ≠ LIG `(공+방)×가용성`** → 논문은 CybORG 기준으로 실험하되, §논의에 *"본선 채점은 가용성 곱셈 → 가용성 보존형 정책이 유리"*를 명시(우리 차별점 연결).

## 7. 작업 분담 제안 (남은 기간)
1. **(즉시)** M1 휴리스틱 강화(Canary/Whistle) + S1~S4 config로 baseline 성능 측정.
2. **(병행)** M2 PPO 학습 파이프라인(SB3) 구축.
3. **(통합)** M3 하이브리드 → 성능 비교표.
4. **(여유)** M4 GNN(범용성 어필).
→ 각 단계 결과를 `analyze.py summary.csv`로 모아 논문 §결과에 직접 투입.

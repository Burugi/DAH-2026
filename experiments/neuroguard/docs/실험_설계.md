# 실험·테스트 설계 — 검증된 설계의 sim 측정 (코드팀 핸드오프 v2)

> **목적:** 기술 검토로 확정된 설계의 핵심 주장을 CybORG/팀레포(`Burugi/DAH-2026`)에서 *측정*해 보고서 §결과로 만든다. **예선 범위 밖(본선 대비) · 코드팀 실행.**
> **선행:** `검토_종합.md`(설계 확정) · `코드팀_구현가이드.md`(S1~S4·M1~M4)
> **원칙:** 잘못된 설계를 테스트하면 환경시간 낭비 → *검토로 확정된 것만* 측정. sim 실현 가능분(A1·A3·A7·복합)만 실험, 나머지는 §위협모델(개념).

## 1. 가설 (검토에서 도출)

| ID | 가설 | 근거(검토) |
|----|------|-----------|

## 4. 측정 지표

- **1차:** 누적 reward · red_owned(감염 수) · **가용성 유지율**(green 성공률) · F1(jam/gps 탐지)
- **2차:** MTTD/MTTR · **과잉대응 횟수**(자해 지표) · gps_err(before→after)
- 다중 seed(≥5) 평균±분산. (검토: 보상 외 MTTD/MTTR·가용성유지율 필수)

## 6. 핵심 실험 3종 (검토 발견 직결 · 보고서 킬러 그래프)

### E1. 모델 비교 (H1·H2)
- M1~M3 × S1~S4, 누적보상·가용성 유지율·과잉대응 횟수.
- **기대:** M3 최상위. 가용성 인식 보상 ON/OFF 대조로 H2(자해 감소) 입증.

### E2. 미지 공격 일반화 (H3) — B20/C16 핵심, 가장 차별적
- **학습:** red 변형 부분집합. **테스트:** held-out 변형(점진드리프트·다른 확산타이밍·노이즈 한도 은밀).
- **측정:** in-distribution vs held-out 탐지율·가용성 유지율 → **일반화 곡선**.
- **정직한 대조(검토 C1):** A19형(노이즈 한도 은밀)은 **L1 단독 탐지율이 낮음을 그대로 보이고**, L2 교차검증(다중센서 일관성)·합의(Canary/Whistle) 추가 시 회복됨을 대조 → "단일 탐지 한계 → 다층 보완" 주장을 *수치로* 입증.

### E3. A3↔B3 결합 취약 재현 (H4)
- 시각 스푸핑 주입 → 기본 B3(GPS시각 타임스탬프) 서명 **우회 성공률** 측정 → **RTC·단조 시퀀스 카운터 보강 후 차단율** 대조.
- **산출:** 결합 킬체인 성립 + 보강 효과 = 공격(A3+A2)·방어(B3 보강) 양쪽 깊이.
- ⚠️ CybORG가 MAVLink 서명·시각을 직접 모델하지 않음 → **fleet.py 합성 신호 레벨에서 추상화로 재현(개념 입증)**, 실 MAVLink는 §위협모델 서술(정직 표기).

## 7. 합격 기준 (설계 검증)

| 기준 | 통과 조건 |
|------|-----------|
| H1 | M3 보상·가용성 ≥ M1·M2 |
| H2 | 가용성 인식 보상 ON이 과잉대응↓ + ×가용성 점수↑ |
| H3 | held-out 가용성 유지율 ≥ 목표치(다층 적용 시), L1단독 대비 다층 우위 |
| H4 | 보강 전 우회 성립 / 보강 후 차단 |
| 결론 | 측정 최상위 모델 = **본선 에이전트** 채택 |

## 9. 리스크·정직 표기

- **추상화 한계:** E3 등은 CybORG 미모델 부분을 합성 신호로 *개념 재현* → 보고서에 "sim 추상화 수준" 명시(실 프로토콜은 위협모델).
- **비용:** 학습 seed 비용 → M3 우선, M4 옵션. E2가 가장 가치 높으니 우선순위 E2≈E1 > E3.
- **순서:** 설계 확정(완료) → E1(기준선) → E2(차별화) → E3(결합) → §결과 투입.

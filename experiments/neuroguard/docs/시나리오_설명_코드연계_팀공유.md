> 지금까지 제목만 있던 시나리오를 **(1) 실제로 무슨 공격인지 (2) CybORG에서 어떤 코드 spec으로 구현됐는지
> (3) 실측 곱셈** 으로 연결. 팀원이 읽고 바로 이해·재현할 수 있게. **단일 진실 = `exp_matrix.py`의 `SCEN` 리스트.**

## 0. 읽는 법 — sim 기제 파라미터 (코드 연계)

23개 시나리오는 실제로 **소수의 sim 기제 파라미터 조합**으로 구현된다. 각 파라미터의 뜻과 코드 위치:

| 파라미터 | 뜻 (실제 공격) | 코드 위치 (`exp_matrix.py`) |
|------|------|------|
| `vectors: W/J/B` | 공격 레인 — **W=웜**(Exploit+전파 `[2,6]`), **J=재밍**(FloodBandwidth `[7,8]`), **B=차단**(BlockTraffic `[9,10]`) | `VEC_AIDS`, `make_red()` |
| `inject` | 감염 드론이 이웃 가용성을 드레인(위조 명령) | `rollout()` 의 `inj` 항 |
| `frag_K` | 상위 K개 허브 제거 = 연결성 분할 | `rollout()`: `hubs = argsort(-deg)[:frag_K]` |
| `blackout_p` | 위성 단절 — p 비율 노드 도달불가 | `rollout()`: `black` |
| `detector_q` | 탐지 저하 — 코디네이터가 감염을 보는 확률 | `rollout()`: `detected` 생성부 |
| `poison_q` | Sybil/합의 오염 — 탐지 회피 확률 | `rollout()`: `detected` 생성부 |
| `tempo` | 은밀/저속 — 매 스텝 행동할 확률 | `make_red()`: `tempo` |
| `start_red` | 내부자 — 초기 감염 드론 수 | `cfg["sim"]["starting_num_red"]` |

> 곱셈종합 = `mean(1−점령, 1−AUC, 0.866) × 가용성`. 표의 값은 **코디네이터 / 통합방어** 기준(같은 seed).

## 2. 재현 방법 (팀원용)

```powershell
# 전 시나리오 × 방어 매트릭스 (표 1의 근거)
& C:\workspace\dah_venv\Scripts\python.exe C:\workspace\DAH2026_exp\exp_matrix.py   # → summary_matrix.csv
# 코디네이터 기준 23종 전수
& C:\workspace\dah_venv\Scripts\python.exe C:\workspace\DAH2026_exp\exp_allscenarios.py
```
- **시나리오 추가/수정:** `exp_matrix.py` 의 `SCEN` 리스트에 `(이름, spec)` 한 줄만 추가하면 됨.
- 하네스: `C:\workspace\DAH-2026\src`(팀 CybORG, **수정 금지**)를 import만. 곱셈·오버레이는 우리 스크립트에서 계산.

## 3. 방어·아키텍처 코드 연계

| 방어/구조 | 코드 |
|------|------|
| 코디네이터(CTDE) | `exp_coordinator.py` `rollout(coord=True)` |
| 통합 방어(분산자율+de-jam+relay) | `exp_ultimate.py` `rollout(mode="ultimate")` |
| 동적 재구성 v2 | `exp_reconfig_v2.py` |
| 탐지 무결성(quorum/봉쇄) | `exp_detintegrity.py` · `exp_containment.py` |
| 임무 무결성 | `exp_mission.py` |

> 이 문서 하나로 "시나리오 제목 → 무슨 공격 → 어느 코드 → 어떤 결과"가 연결된다. 노션에도 동일 표를 올릴 수 있음.

## A1. 공급망 펌웨어 트로이목마 → 군집 웜 확산 ★대표 시나리오 [sim]

- **근거** — UAV는 공급망(부품·펌웨어·OTA) 신뢰가 핵심인데 검증이 취약. CAGE CC3가 바로 이 위협 모델(드론 군집 + 펌웨어/하드웨어 트로이목마 + 웜)로 검증됨 → 방산 현실성 + 벤치마크 권위 동시 확보.
- **메커니즘** — 펌웨어/부트로더에 휴면 악성코드 삽입 → 트리거(시간/명령/위치) 시 활성화 → MANET을 통해 인접 드론으로 웜 확산 → 정보 탈취 또는 허위 텔레메트리 주입.
- **킬체인(ATT&CK)** — Supply Chain Compromise(T1195) → Firmware 지속(T1542) → 횡적 이동(군집 전파) → 데이터 조작(T1565)/정보수집.
- **AI 공격 에이전트 적용** — 확산 타이밍·대상 드론을 자율 선택(탐지 회피 위해 점진 확산), 단절 구간을 노려 활성화.
- **가용성 고려** — 즉시 추락시키지 않고 은밀 잠복 + 선택적 정보 유출 → 가용성 점수 유지하며 공격 점수 획득(가장 교과서적 고득점형).

## A2. GCS 침투 → MAVLink 명령 주입 → 기체 탈취 [개념]

- **근거** — GCS(지상국)는 보통 일반 IT라 침투가 상대적으로 쉽고, 장악 시 합법 채널로 명령을 내릴 수 있어 탐지가 어렵다. MAVLink 1.0 평문 취약성과 결합.
- **메커니즘** — GCS PC 침투(피싱/취약점) → MAVLink 세션 장악 → COMMAND_LONG 위조로 항로 변경/무장/RTL 강제 → 기체 탈취 또는 임무 무력화.
- **킬체인** — Initial Access(T1190/피싱) → GCS 제어 → 명령 주입/위조(T1565) → Impact(T1489 임무 거부).
- **AI 적용** — 정상 운용 패턴을 학습해 그 틈에 위조 명령을 섞어 IDS 회피(behavior blending).
- **가용성** — 미세 항로 변경 등 저강도 지속 조작으로 SLA 유지하며 임무 효과만 훼손.

## A3. GPS 스푸핑 → 항법 기만 → 경로 이탈 / 지오펜스 우회 [sim]

- **근거** — 민간 GNSS는 비암호화 → 스푸핑 고전적·강력. 물리적 납치(경로 유도)까지 가능해 방산 위협의 대표.
- **메커니즘** — 가짜 GNSS 신호로 위치/시간 기만 → 기체가 잘못된 위치로 비행 → 지오펜스 우회·유도 착륙·경로 이탈.
- **킬체인** — 신호 정찰 → 점진 오프셋 스푸핑(급격하면 INS 불일치로 탐지) → 목표 좌표로 유도.
- **AI 적용** — INS 교차검증을 회피하도록 오프셋 증가율을 동적 조절(탐지 임계 밑으로).
- **가용성** — 기체는 계속 비행(가용성 유지) → 임무 위치/효과만 왜곡 → 공격 점수 + 가용성 동시 확보.
- **결합 강화(A3→B3 우회, A2 연계)** — RTC 없는 기체는 GPS 시각으로 MAVLink 2.0 서명 타임스탬프를 갱신함 → **시각 스푸핑이 서명 anti-replay를 우회**(ArduPilot #13860, IEEE 2024). A3로 B3 서명 방어를 무력화한 뒤 A2 재전송으로 명령 주입 → 결합 킬체인(보고서 차별 포인트).

## A4. 위성링크(SATCOM) 공격 — 지연/단절 악용 + C2 MITM·재전송 [개념]

- **근거** — 대회 환경이 위성망 모사 → 이 시나리오가 가장 대회 맞춤. 위성 링크의 고지연·간헐 단절은 고유 공격면.
- **메커니즘** — C2 위성링크에 MITM/재전송(replay) → 과거 유효 명령 재생, 또는 단절 구간에 명령 위조 삽입. 링크 미암호화/약한 인증 가정.
- **킬체인** — 링크 트래픽 정찰 → 재전송 가능 메시지 수집 → 단절·핸드오버 타이밍에 주입.
- **AI 적용** — 단절 패턴을 예측해 관제 사각 구간에 자율 행동(사람 개입 어려운 시점 공략).
- **가용성** — 링크 자체는 살려두고 명령 무결성만 훼손 → 가용성 곱셈 방어.

## A5. 컴패니언 컴퓨터 / ROS2(DDS) 미들웨어 익스플로잇 [개념]

- **근거** — AI·영상처리용 컴패니언(Jetson 등)은 ROS2/DDS 사용. SROS2 미적용 시 토픽 인젝션·노드 탈취 용이. UGV/고급 UAV에 직접 해당.
- **메커니즘** — DDS 디스커버리 악용 → 악성 노드 합류 → 센서/명령 토픽에 위조 데이터 주입 → 자율 판단 왜곡(페이로드·항법).
- **킬체인** — 원격 서비스 익스플로잇(T1210) → 토픽 인젝션(데이터 조작 T1565) → 자율 스택 오작동.
- **AI 적용** — 어떤 토픽을 노릴지(최대 효과/최소 노출) 자율 선택.
- **가용성** — 핵심 제어 토픽이 아닌 인지/판단 토픽을 미세 왜곡 → 시스템 동작 유지하며 임무 품질 저하.
- **방어 매핑** — SROS2 노드 인증·토픽 ACL + B11(제로트러스트·마이크로세그멘테이션). *(검토 보강: 기존 매핑 누락분)*

## A6. 센서 스푸핑 (UGV LiDAR/카메라/IMU) [개념]

- **근거** — UGV는 센서 의존도 극대 → 센서 기만이 치명적. 적대적 예제(카메라) + LiDAR 스푸핑.
- **메커니즘** — 가짜 장애물/유령 물체 주입(LiDAR), 표지판/객체 오인식(카메라 adversarial), IMU 진동 주입.
- **킬체인** — 센서 입력 조작 → SLAM/인식 오류 → 정지·우회·충돌 유도.
- **AI 적용** — 환경에 맞춰 가장 효과적인 스푸핑 패턴 생성(상황 적응).
- **가용성** — 차량 정지(가용성↓) vs 미세 경로 왜곡(가용성 유지) 중 점수 최적점을 에이전트가 선택.

## A7. RF 재밍 / 미코닝(meaconing) [sim]

- **근거** — 전자전(EW)에서 가장 흔한 실전 위협.
- **메커니즘** — 광대역·반응형 재밍으로 통신 두절(DoS), 미코닝=정상 신호 재방송 기만. 위성망 단절과 결합.

## A8. C2 링크 탈취 / 디오쓰(deauth) [개념]

- **근거** — 상용 드론 무선 탈취 공개 데모 다수.
- **메커니즘** — 텔레메트리 무선 디오쓰 → 세션 가로채기 → 조종권 탈취.

## A9. OTA 펌웨어 업데이트 악용 [개념, A1 변형]

- **근거** — 공급망 공격의 운영단계 연장.
- **메커니즘** — 위조 펌웨어 푸시·다운그레이드(롤백)로 취약 버전 강제.

## A10. 스웜 특화 — Sybil·비잔틴 교란 [개념, MANET]

- **근거** — 군집 합의·라우팅의 구조적 취약.
- **메커니즘** — 가짜 ID 다수 생성 → 합의 왜곡·블랙홀/웜홀 라우팅 → 군집 분열.

## A11. 방어 AI 자체 공격 (Adversarial ML) [개념, 차별적]

- **근거** — 우리가 AI 방어를 쓰면 그 AI가 표적(메타 위협).
- **메커니즘** — 탐지 회피(evasion)·모델 포이즈닝·적대적 텔레메트리로 오탐 유발. → ②의 한계를 스스로 짚는 고급 논점.

## A12. 임베디드 사이드채널 [개념]

- **근거** — 물리 접근 가능한 무인체계의 임베디드 특성.
- **메커니즘** — 전력분석·타이밍으로 키/로직 추출, 글리치 공격.
- **검토 판정** — 물리 계측 기반 → 클라우드 sim·자율공방 범위 밖. **'물리보안 영역'으로 각주화 또는 제외 권장**(대응 방어도 부재).

## A13. PNT/시간동기 공격 [개념, 위성망 특화]

- **근거** — 위성망·항법은 GPS 시간동기 의존.
- **메커니즘** — 시간 스푸핑으로 타임스탬프·로그·재전송 방어 교란.

## A14. 다영역 협조 공격 [sim 일부]

- **근거** — 실전 공격은 동시다발.
- **메커니즘** — 재밍(은폐)+웜(침투)+GPS(유도)를 시간 조율 → 단일 방어로 못 막게.

## A15. 내부자 위협(Insider) [개념]

- **근거** — 운용자·정비 인력의 합법 접근 악용(탐지 최난).
- **메커니즘** — 정상 권한으로 펌웨어·미션 설정 변조·정보 유출.
- **검토 판정** — UAV 도메인 특수성·자율성 어필 약함 → **인사·운영 보안 영역으로 축소**(대응: 제로트러스트·최소권한·감사로그 B16).

## A16. ISR 임무데이터 탈취(Exfiltration) [개념]

- **근거** — 무인체계의 핵심 가치 = 수집 정보.
- **메커니즘** — 페이로드 영상/표적 데이터를 저속 채널·스테가노그래피로 은밀 유출.
- **방어 매핑** — 페이로드 데이터 암호화·DLP(유출방지) + B17 키관리. *(검토 보강: 기밀성 방어 누락분)*

## A17. 스웜 지휘체계 탈취(C2 takeover) [개념, A10 상위]

- **근거** — 군집 조정/리더 노드 장악 시 편대 전체 통제.
- **메커니즘** — 리더 선출·조정 노드 침해 → 편대 재지정·임무 변경·자폭 유도.

## A18. SwarmFuzz — 단일 노드 스푸핑으로 군집 연쇄 충돌 [논문심화]

- **근거** — 군집 중 단 한 대에만 GPS 스푸핑을 가해도 협조 제어 알고리즘 상호작용으로 나머지 드론이 연쇄 경로 이탈·충돌(SwarmFuzz, DSN 2023, 코드 공개).
- **공격 형태** — 그래프 이론 + gradient-guided 최적화로 "어느 한 대를 어떻게 흔들면 군집이 무너지는지" 공격 파라미터를 자동 탐색(무작위 대비 약 10배 성공). 군집이 클수록 더 취약.
- **검토 의견** — '1점 공격 → 군집 전파'의 강력한 메시지 + A1·A10과 시너지.

## A19. Raven — 센서 노이즈 한도 내 은밀 의미론 공격 [논문심화]

- **근거** — 군집 항법을 무너뜨리되 주입값을 센서 노이즈 범위 안 + 시공간 일관성으로 유지해 임계 기반 탐지를 회피(Raven, USENIX Security 2025).
- **공격 형태** — 시간논리 + 확률최적화로 '최소 변위·최소 주입'의 은밀 공격을 자동 탐색(단 1회 주입으로도 군집 붕괴).
- **검토 의견** — "왜 단순 임계 탐지만으론 부족한가"의 직접 근거 → 방어(카이제곱/CUSUM) 고도화 명분.

## A20. Incalmo — LLM 기반 kill-chain 자율 실행 [논문심화]

- **근거** — LLM 공격 에이전트가 "pivot → 자격증명 탈취 → 정보 유출" 같은 선언적 고수준 태스크만 받아 다중호스트 공격 체인을 자율 실행(Incalmo, arXiv:2501.16466, 40개 환경 중 37개 성공).
- **공격 형태** — attack-graph가 환경 상태로부터 취약점 연쇄 경로를 동적 추천 → 추상화 계층이 실제 도구로 번역.
- **검토 의견** — 공격 에이전트의 '정해진 행동, 무한한 조합' 자율 연쇄를 학술적으로 뒷받침.

## A21. VLM/VLA 프롬프트 인젝션 — 방어 AI 인지층 탈취 [논문심화, A11 심화]

- **근거** — ISR 영상·표적식별에 시각-언어 모델(VLM)을 쓰면, 장면 속 이미지에 은닉(스테가노)한 명령으로 에이전트의 인지·행동을 통째로 탈취 가능(arXiv:2507.22304 외 — 여러 SOTA VLM에서 취약 보고).
- **공격 형태** — cross-modality 자극으로 모델 정렬(alignment)을 깨뜨려 시각 입력을 덮어쓰는 행동 붕괴.
- **검토 의견** — "방어가 AI면 그 AI가 곧 표적"이라는 메타 논점 → ③ 에이전트의 자기위협 분석으로 차별화.

## B1. 텔레메트리 / 행위 이상탐지 (시계열 ML) — 대응: A2,A3,A4,A7 [sim]

- **근거** — 위조 명령·스푸핑은 정상 패턴에서 미세 이탈을 남김. 임베디드 텔레메트리(자세·고도·전류·링크품질)는 시계열 이상탐지의 좋은 입력.
- **메커니즘** — 경량 시계열 모델(LSTM-AE, Isolation Forest) + 규칙으로 이상 점수화 → 임계 초과 시 경보.
- **가용성** — 탐지 자체는 비파괴적이나, **오탐이 B7/B19 대응 사다리를 발동시키면 간접 자해** → 가용성은 상류 탐지 정확도에 종속. 따라서 green 노이즈 모델로 오탐 억제 + 저신뢰 시 보수적(저침습) 대응이 전제.

## B2. GPS 스푸핑 탐지 (센서 퓨전 교차검증 + RAIM) — 대응: A3 [sim]

- **근거** — INS와 GNSS의 물리적 일관성을 깨면 스푸핑 탐지 가능. RAIM·다중 GNSS·도래각.
- **메커니즘** — INS 추정 위치 vs GNSS 위치 잔차 모니터링 + 신호 파워/도래각 이상 + RAIM. 불일치 시 INS 기반 추측항법으로 페일세이프.
- **가용성** — GNSS 의심 시에도 INS로 비행 유지 → 가용성 보존하며 공격 무력화(핵심 우승 포인트).

## B3. MAVLink 무결성 / 인증 — 대응: A2,A4 [개념]

- **근거** — 평문 MAVLink가 주입/재전송의 근본 원인 → 서명·시퀀스·논스로 차단.
- **메커니즘** — MAVLink 2.0 메시지 서명 활성화, 메시지 카운터/타임스탬프로 재전송 거부, 명령 화이트리스트·레이트리밋, 비정상 명령 시퀀스 탐지.
- **⚠️ 한계·보강** — GPS 시각 의존 타임스탬프는 시각 스푸핑(A3)으로 우회됨(ArduPilot #13860) → **독립 RTC + 단조 시퀀스 카운터**로 anti-replay를 GPS 비의존화해야 A3 결합 공격을 막는다.
- **가용성** — 위조만 거부하고 정상 명령은 통과 → 가용성 영향 최소.

## B4. 네트워크 분할 + IDS + 존 트래픽 제어 — 대응: A2,A5 [개념]

- **근거** — CAGE CC4의 BlockTrafficZone/AllowTrafficZone(존 단위 방화벽 제어)이 검증된 방어 행동. 횡적 이동·확산 차단.
- **메커니즘** — GCS·컴패니언·FCU를 존으로 분할, IDS로 이상 플로우 탐지, 의심 존만 선택 차단(전면 차단 ✗).
- **가용성** — 핵심 제어 경로는 유지하고 의심 경로만 차단 → 가용성 보존형.

## B5. Canaries & Whistles — 군집 감염 탐지·전파 — 대응: A1 [CC3 우승]

- **근거** — CC3 우승작(휴리스틱이 PPO를 약 4~5배 앞섬: −1577 vs −7617). A1(군집 웜)에 대한 직접 대응책으로 강력하고 구현 단순.
- **메커니즘** — 각 정상 드론이 주기적 ID 하트비트(Canary) 브로드캐스트 → 멈추면 감염 추정. 감염 ID를 군집 전체로 재전파(Whistle) → 이웃이 해당 노드 격리/불신.
- **가용성** — 감염 의심 노드만 격리, 군집 임무는 지속 → 가용성 유지하며 확산 차단. 고득점·고현실성·저구현비용의 삼박자.

## B6. 기만(Deception) — 디코이 / 허니팟 — 대응: A1,A2,A5 [CC2 우승]

- **근거** — CAGE CC2 우승 레시피 = PPO + 탐욕적 디코이. 공격자를 가짜로 유인해 조기 탐지 + 시간 소모.
- **메커니즘** — 가짜 서비스/가짜 드론/허니 토픽 배치 → 공격자가 건드리면(정상 사용자는 안 건드림) 명백한 침해 신호.
- **가용성** — 디코이는 실제 서비스와 분리 → 가용성 무손실 + 탐지력↑.

## B7. 가용성 보존형 자율 대응 (대응 정책의 핵심) — 대응: 전체 [sim]

- **근거** — ×가용성 채점의 직접 대응. CAGE의 Restore(복원)도 페널티(운영중단 비용)라는 교훈 → "끄고 재시작"은 최후수단.
- **메커니즘** — 대응 단계화: ①모니터링 강화 → ②격리/세분 차단 → ③페일세이프(INS항법·안전모드) → ④(최후)복원/재시작. 에이전트가 위협 심각도 vs 가용성 손실을 저울질해 최소침습 대응 선택.
- **가용성** — 설계 자체가 가용성 최적화 → 본선 점수 직결.

## B8. 시큐어부트 + 펌웨어 무결성 원격증명(attestation) [개념]

- **근거** — A1/A9 공급망의 정공법 대응. 메커니즘: 서명 검증 부트체인 + TPM/원격 증명으로 변조 탐지.

## B9. 침입감내 / 비잔틴 내성 군집 합의 [개념, B5 연계]

- **근거** — 일부 노드 감염돼도 임무 지속=가용성 보존(채점 직결). 메커니즘: BFT 합의·신뢰가중 라우팅으로 감염 노드 영향 격리.

## B10. 무빙타겟 방어(MTD) [개념]

- **근거** — 공격면을 동적 변경해 정찰·지속을 무력화. 메커니즘: 주소/포트/주파수 도약, 키 롤링. ⚠️ 위성 고지연 환경에선 도약이 green 트래픽을 끊어 가용성 자해 위험 + CC3엔 정찰단계가 없어 주효과 미발현 → **본선 한정·저빈도·핵심경로 보존 조건부** 적용.

## B11. 제로트러스트 · 마이크로세그멘테이션 [개념, B4 연계]

- **근거** — 웜 횡적 확산 차단. 메커니즘: 노드 간 상호 인증·최소권한, 세그먼트별 정책.

## B12. RF 핑거프린팅 [개념, B2 연계]

- **근거** — 송신기 하드웨어 물리특성으로 위장/스푸퍼 식별. 메커니즘: RF 지문 분류로 정상 송신원 검증.

## B13. 연합학습 기반 분산 이상탐지 [개념]

- **근거** — 스웜이 경험 공유하되 대역폭·프라이버시 보존. 메커니즘: federated anomaly 모델(노드 로컬 학습+집계).

## B14. 방어 AI의 적대적 견고성 [개념, A11·A21 대응]

- **근거** — 방어 AI가 표적될 때 자기보호. 메커니즘: adversarial training·입력 검증·앙상블·불확실성 추정.

## B15. 디지털 트윈 / 물리모델 예측 탐지 [개념, B2 일반화]

- **근거** — 물리 일관성은 위조 난해. 메커니즘: 물리모델 예측 vs 텔레메트리 잔차로 스푸핑·이상 탐지.

## B16. 로깅·포렌식·사후 IR [개념]

- **근거** — 탐지 후 원인분석·복구·재발방지(가용성 회복 단계). 메커니즘: 변조방지(append-only) 로그, 타임라인 재구성, 대응 플레이북.

## B17. Fleet 키관리·PKI [개념, B3·B8 토대]

- **근거** — B3 서명·B8 attestation의 신뢰 뿌리(루트). 메커니즘: 기체별 키/인증서 발급·폐기(rotation), HSM/TPM 보관.

## B18. 자원고갈(DoS) 방어 [개념]

- **근거** — 임베디드 자원 제약 → 연산·대역폭 고갈 공격에 취약. 메커니즘: 레이트리밋·우선순위 큐·연산예산 가드(가용성 직접 보호).

## B19. 공격유형별 차등 복구 (가용성 인식 RL) — 횡단 원칙 ★핵심 [논문심화]

- **근거** — 채점식 (공격+방어)×가용성에서 과잉 대응(전면 차단·재시작)은 가용성 손실로 자해. 공격 유형에 따라 복구 깊이를 차등화: 스푸핑→상태추정 복원(deep) / DoS→채널 전환(shallow) / 노드탈취→합의 재선출. (CPS 복구 taxonomy 연구공백 보완, arXiv:2404.04472)
- **메커니즘** — 대응 사다리 ①모니터링 → ②선택적 격리 → ③페일세이프(INS) → ④(최후)복원을 가용성 인식 보상으로 학습. 보상 '과잉대응 −10 / 정지 −1 / 임무 +50'으로 보안-가용성 균형 직접 최적화(FFI 실사례 arXiv:2410.21407). B1~B7의 '복구' 단계를 관통하는 상위 원칙.

## B20. 미지 공격(처음 보는 공격) 대응 4계층 프레임 — 견고성×범용성 ★핵심 [통합]

> 공격을 *이름으로 식별*하지 않고, *정상에서의 이탈을 효과로 분류 → 효과 기반 등급 대응 → 학습*한다. 본선 상대의 창의적·미지 공격을 전제로 한 방어 견고성의 중심 프레임.

- **문제** — 본선 상대는 창의적 → signature(알려진 공격 목록) 기반 방어는 새 공격에 반드시 뚫림. 미지 공격에 과잉대응하면 ×가용성 자해. 본선 action space도 다를 수 있음 → 전용 규칙은 이식 불가.
- **L1 탐지(signature-free)** — 불변식 위반 감지: 물리(INS↔GPS 잔차·동역학 한계)·프로토콜(MAVLink 시퀀스·DDS 권한·명령 레이트)·행동(LSTM-AE 재구성오차·CUSUM)·OOD(모델 신뢰도 급락). 많은 공격은 불변식을 깨지만, **노이즈·일관성 한도 내에 머물도록 설계된 은밀 공격(A19 Raven·A3 점진 드리프트·A21)은 L1 단독으로 못 잡음을 인정** → L2·L4가 메운다(단일 탐지의 한계를 알고 다층 설계).
- **L2 분류(효과 택소노미)** — 미지 공격을 5버킷으로 환원: 항법 이상 / 통신 이상 / 제어 이상 / 노드 이상 / 인지 이상 → 알려진 대응 카테고리로 연결(식별 불가 ≠ 대응 불가). 단 A1 웜·A14 다영역은 다중 버킷 동시 발현 → **복합 라우팅·우선순위 규칙** 필요(완전 MECE 아님). 은밀 공격은 L2에서 **교차모달 검증(다중센서·VLM 일관성)**으로 보강.
- **L3 대응(가용성 인식 등급)** — 사다리 ①모니터링↑ → ②선택적 격리 → ③페일세이프(INS/안전모드/채널전환) → ④(최후)복원·재선출. 버킷별 차등 복구(B19 연계). 불확실할수록 보수적 대응(과잉대응 페널티 회피) — **단 확산성 공격(A1 웜)엔 보수=방치 자해이므로 '불확실성 × 잠재확산피해' 2축으로 대응 강도 결정.**
- **L4 적응(미지→기지)** — 변조방지 로그 → 사후 분석 → 패턴 라이브러리 추가·재학습 → 같은 공격 재발 시 즉시 기지 대응으로 강등.
- **횡단 가드** — hard shield(green 차단 금지·비행 중 핵심 재시작 금지·물리 안전 한계 = 진짜 미지 공격의 최후 안전망) + 범용성(효과↔대응 의미 추상화 + GNN 노드함수로 토폴로지·군집 크기 변화에 강함=이식성). ⚠️ GNN은 *이식성* 근거이지 성능 우위가 아님(CC4서 GNN −193 < 휴리스틱 −118) → 성능 주력은 휴리스틱+PPO.
- **점수 연결** — 견고성(25pt): 목록에 없는 공격까지 흡수. 가용성: 보수적 대응으로 자해 방지. 범용성: 본선 action 변경에도 이식. 공격(창의)↔방어(견고)의 대칭 서사.
- **킬러 실험** — 학습에 없던 red 변형(미지 공격 대용)에 대한 탐지율·가용성 유지율 = '일반화 곡선' 그래프 → 견고성·범용성을 수치로 증명. *(상세 설계: `미지공격_대응아키텍처` 문서. B7·B14·B19·C3·C8을 묶는 상위 통합 프레임.)*

## C1. 전체 아키텍처 — 3-에이전트 정식화 + OODA 루프

- **설계 철학** — Blue(방어)/Red(공격)/Green(정상 임무 트래픽=가용성 기준)의 3-에이전트 정식화(검증된 틀). UAV/UGV엔 사람 '사용자'가 없으므로 CybORG의 green을 '드론의 정상 통신·임무 수행'으로 재정의(가용성 측정 기준).
- **공용 골격(Blue/Red 공통)** — 인지(Perception: 관측 정규화·특징추출·상태추정 POMDP) → 추론(Reasoning: LLM 에이전틱 상황해석·계획·도구선택) → 결심(Decision Core: 휴리스틱 + RL(PPO) + 안전가드) → 행동(Action: MAVLink/존차단 등 실행). Memory/World Model이 과거 관측·공격 이력 보존.
- **OODA 대응** — Observe(정찰·모니터) · Orient(위협 분석) · Decide(대응 기획) · Act(복구·실행). 사람 개입 없는 자율 인지·대응.

## C2. 정식화 (관측 · 행동 · 보상)

- **관측(POMDP)** — 텔레메트리(자세·고도·링크품질), 네트워크 플로우, 시스템/ROS2 로그, IDS 알람, GNSS/INS 잔차 → 정규화·특징벡터화(CybORG BlueTableWrapper 참조).
- **행동(Action)** — Blue: Monitor / Analyse / Isolate(격리) / Restore(복원) / DeployDecoy / BlockZone / Failsafe(INS전환). Red: Recon / Exploit / Inject / Propagate / Persist / Impact.
- **보상(가용성 인식형) ★핵심 혁신** — R_blue = Σ(위협차단 가점) − Σ(침해 손실) − λ·(가용성 손실) · R_red = Σ(공격목표 달성) − μ·(노출·탐지 위험). ⚠️ 가용성은 채점식에만 곱셈, 에이전트 보상은 가중합(곱셈은 reward hacking/그래디언트 불안정, 검증 반영).

## C3. 의사결정 코어 — 하이브리드 (왜 이렇게?)

- **(a) 도메인 휴리스틱** — Canary/Whistle, RAIM 임계, 명령 화이트리스트 등 검증된 규칙 → 즉시 동작·해석가능·경량(임베디드 적합). CAGE에서 휴리스틱이 RL을 이긴 근거.
- **(b) RL(PPO)** — 휴리스틱으로 안 잡히는 미묘한 상황의 정책 최적화. DQN 회피(부분관측·노이즈 불안정), Stable-Baselines3 사용.
- **(c) LLM 에이전틱 추론** — novel·복합 상황 해석, 다단계 계획, 도구 선택. 단 추론루프에선 빼고 오프라인으로(보상설계·PPO 부트스트랩·incident 설명). 8B=3.69s/step → 임베디드 실시간 불가.
- **(d) 안전 가드(hard shield)** — 어떤 정책도 가용성·안전 제약 위반 행동은 차단(비행 중 전체 재시작 금지 등). 계층형 = 상위 LLM 플래너 + 하위 휴리스틱/RL 실행기.

## C4. 학습 전략

- **자가대전(Self-play)** — Blue↔Red 동시 강화로 강건성.
- **커리큘럼** — 단순 공격(A2)→복합(A1 군집)으로 난이도 상승.
- **Green 시뮬레이션** — 정상 트래픽 주입으로 오탐 억제·가용성 측정 현실화.
- **sim-to-real** — 임베디드 자원/지연 제약 반영 + domain randomization + teacher-student distill(LLM teacher→경량 RL student 임베디드 배포).

## C5. 혁신 포인트 (보고서 강조)

1. 가용성 인식 보상함수 — 대회 채점을 에이전트 목표로 내재화(직접적·논리적).
2. 휴리스틱+RL+LLM 3중 하이브리드 — CAGE 교훈 기반, 해석가능성+적응성+계획성.
3. 임베디드 sim-to-real — "게임을 넘어 실전"(CAGE의 공인된 한계를 메움).
4. Blue/Red 공용 골격 + self-play — 공격·방어를 한 프레임워크로(설계 일관성).

## C8. GNN 기반 정책 (그래프 표현) [대안]

- **근거** — 군집/네트워크는 본질적으로 그래프. 노드=드론/호스트·엣지=링크, 행동을 노드 단위 함수로 → 네트워크 크기·미관측 토폴로지 변화에 강함(이식성, Cybermonic GNN+PPO). ⚠️ 단 *성능* 우위는 아님(CC4서 GNN −193 < 휴리스틱 −118) → **'범용성 보조'로 활용, 성능 주력은 휴리스틱+PPO**.

## C9. 계층형 MARL (CTDE) [대안]

- **근거** — CC3는 드론당 blue 1개(분산 방어). 중앙학습·분산실행(MAPPO) + 상위 조정자. 단 무작위 초기화 적응·파라미터 공유 병목 주의.

## C10. Decision Transformer / 시퀀스 모델 [대안]

- **근거** — 오프라인 RL·임베디드 경량. 과거 (관측·행동·보상) 시퀀스로 행동 예측 → 자원 제약 유리.

## C11. 앙상블 정책 [대안]

- **근거** — 단일 정책 과적합 회피, CAGE에서 앙상블이 일반화 우수. 다수 정책 투표/선택.

## C12. LLM-as-Judge / 설명가능 계층 [대안]

- **근거** — 사람이 읽는 incident 판단·심사 보조. 추론루프 밖 사후 설명(데모·보고서 가치).

## C13. 월드모델 기반 (model-based RL) [대안]

- **근거** — 동역학 학습으로 샘플효율↑·위협 전개 선제 예측.

## C14. 뉴로심볼릭 (규칙+학습) [대안]

- **근거** — 도메인 규칙(MAVLink 검증·RAIM)은 심볼릭, 미세조정은 학습 → 해석가능성+적응성.

## C15. 벤치마크 · 구현 정합 (CAGE/CybORG + 코드팀 확정) [정합]

- **벤치마크** — 자체 환경 발명 대신 기존 CybORG(CAGE) 표준 채택. LIG의 (공+방)×가용성 환경 직접 반영은 환경 대수정 필요 → 미채택. 공식 CAGE 논문 레퍼런스, CybORG 시각화 재사용.
- **Action space** — 자체 20개 발명 대신 CAGE CybORG 소수(약 8개) 행동 채택(Monitor/Analyse/Remove/Restore/Decoy/Block/Allow/Sleep). '정해진 행동, 무한한 조합'으로 자유도 통제, 본선 행동 변화 대비 범용성(의미단위 추상화·GNN) 확보.
- **모델 비교(핵심 실험)** — 정해진 행동 내 M1 휴리스틱 · M2 PPO · M3 하이브리드(주력) · M4 GNN 동일 조건 비교. 근거: CAGE 상위 1~3위가 휴리스틱 → 규칙기반 + RL 하이브리드가 우승 공식.
- **결론** — 정해진 행동 집합 내 최고 성능 모델(누적보상·가용성 유지율)을 본선 에이전트로 채택(예상: 하이브리드 최상위).

## C16. 미지 공격 대응 에이전트 흐름 (OODA 실현) ★핵심 [B20 실현]

- **데이터 흐름** — Observe(L1 불변식 모니터: 물리·프로토콜·행동·OOD) → Orient(L2 효과 버킷 분류 + 불확실성 추정) → Decide(L3 등급 대응 = 효과 깊이 × 가용성 가중) → Act(격리/INS/마스킹/재선출) → L4 로깅·오프라인 적응 → 루프.
- **LLM 오프라인 전용** — 미지 패턴 사후 해석·새 룰 제안·incident 설명만 담당(실시간 결정 루프 미투입, 지연 제약).
- **C3 하이브리드 코어 보강** — 휴리스틱(action-mask)+PPO+안전가드 코어는 유지하되, 본 설계가 그 *입력단(탐지·분류)과 적응단*을 미지 공격에 견고화. 방어 측 전체 프레임은 ② B20 참조.

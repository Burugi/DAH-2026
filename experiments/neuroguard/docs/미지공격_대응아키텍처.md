# 처음 보는 공격(미지 공격) 대응 아키텍처 — 견고성 × 범용성

Team NeuroGuard · DAH 2026 · ② 방어(견고성) + ③ AI 에이전트(혁신성) 보강 설계

> **한 줄 요약:** 공격을 *이름으로 식별*하지 않고, *정상에서의 이탈을 효과로 분류 → 효과 기반 등급 대응 → 복원하며 학습*한다. → 목록에 없는 공격도 막고(견고성), 본선 행동집합이 바뀌어도 이식된다(범용성).

---

## 0. 왜 이 설계가 필요한가 (문제 정의)

1. **본선 상대는 창의적이다.** 알려진 공격 목록(signature) 기반 방어는 *새로운 공격에 반드시 뚫린다*. 공격 배점(30pt)의 핵심이 '창의성'이므로, 방어는 *처음 보는 공격*을 전제로 설계해야 한다.
2. **과잉대응 = 자해.** 채점 `(공격+방어)×가용성`에서 미지 공격에 놀라 전면 차단·재시작하면 가용성이 0에 수렴 → 점수 폭락.
3. **본선 action space가 예선과 다를 수 있다.** "이 공격엔 이 행동" 식의 전용 규칙은 이식되지 않는다 → 의미 단위 추상화가 필요.

→ 결론: 방어 에이전트는 **(불변식 기반) 탐지 → (효과 기반) 분류 → (가용성 인식) 등급 대응 → (학습 기반) 적응**의 4계층 + 횡단 안전가드 구조여야 한다.

---

## 1. 설계 원칙 4가지

| # | 원칙 | 의미 |
|---|------|------|
| P1 | **Signature가 아니라 Invariant(불변식)** | 공격 벡터가 무엇이든 물리·프로토콜·행동 *불변식*은 깨진다. 알려진 패턴이 아니라 "정상의 위반"을 잡는다. |
| P2 | **식별(identity)이 아니라 효과(effect)** | 미지의 공격도 관측 가능한 *증상 소수 범주*(항법/통신/제어/노드/인지)로 매핑 → 알려진 대응으로 연결. |
| P3 | **효과 기반 등급 대응 + 가용성 인식** | 증상 심각도에 비례한 최소침습 대응. 불확실할수록 보수적(과잉대응 회피). |
| P4 | **미지 → 기지 전환(적응)** | 새 패턴을 로깅·분석·재학습 → 다음엔 "알려진 공격"으로 강등. |

## 2. 4계층 구조 (+ 횡단 가드)

```
 [환경 관측]
     │
 L1 탐지   ── 불변식 기반 이상 탐지 (signature-free)        "정상이 아니다"
     │
 L2 분류   ── 효과(증상) 택소노미 5버킷 매핑 + 불확실성       "어떤 증상인가"
     │
 L3 대응   ── 효과 기반 등급 대응(가용성 인식 사다리)         "얼마나 개입할까"
     │
 L4 적응   ── 로깅 → 사후 분석 → 재학습(미지→기지)            "다음엔 안 당한다"
     │
 ─ 횡단 ─  안전 가드(hard shield) · 범용성(의미추상화·GNN)
```

### L1. 탐지 — 불변식 기반 이상 탐지 (signature-free)

공격의 *종류*를 몰라도, **많은** 공격은 아래 불변식 위반을 남긴다. (단, 노이즈·일관성 한도 안에 머물도록 설계된 은밀 공격은 회피 가능 → L2·L4가 보완)

- **물리 불변식:** INS↔GPS 잔차, 동역학 한계(가속·선회율), 추력/에너지 일관성 → *급격한* 항법 교란은 물리 위반을 남김(점진 드리프트·A3-slow는 회피 가능).
- **프로토콜 불변식:** MAVLink 시퀀스/논스, DDS 토픽 발행 권한, 명령 레이트 한계 → 주입·재전송은 프로토콜 위반.
- **행동 baseline:** LSTM-AE 재구성오차, CUSUM 변화점 탐지 → 정상 임무 분포에서의 이탈.
- **OOD/불확실성:** 모델 신뢰도(엔트로피·앙상블 분산) 급락 = "처음 보는 입력" 자체의 신호.

> 핵심: "이게 무슨 공격인가"가 아니라 **"정상이 아니다"**를 잡는다. → 미지 공격 *대부분*을 탐지.
>
> **⚠️ L1의 한계(정직):** A19(Raven)·A21·A3 점진드리프트처럼 *정상 분포·노이즈 한도 안에 머물도록 설계된* 은밀 공격은 L1을 의도적으로 회피한다. 이는 L2 교차모달 검증(다중센서·VLM 일관성) + 합의 기반 교정(B5 Whistle·B9 BFT·SwarmRaft) + L4 적응으로 메운다. → "단일 탐지에 의존하지 않는 다층 설계"가 본 프레임의 핵심.

### L2. 분류 — 효과(증상) 택소노미로 매핑

미지의 공격 → 관측 효과로 **5개 버킷** 중 하나(또는 복합)로 환원.

| 효과 버킷 | 관측 신호 | 매핑되는 공격(미지 포함) |
|-----------|-----------|--------------------------|
| **항법 이상** | 위치/헤딩 드리프트, INS잔차↑ | GPS 스푸핑(A3)·미지 항법 교란 |
| **통신 이상** | 링크품질↓·재전송·지연 급증 | 재밍(A7)·MITM(A4)·미지 통신공격 |
| **제어 이상** | 비정상 명령/구동 로짓 | MAVLink 주입(A2)·ROS2(A5)·미지 제어공격 |
| **노드 이상** | 세션·호스트 장악 징후, 군집 침묵 | 웜(A1)·탈취(A8/A17)·미지 침투 |
| **인지 이상** | 센서/비전 출력 불일치 | 적대 패치·센서 스푸핑(A6)·미지 인지공격 |

> 새 공격이라도 *효과*는 이 중 하나로 떨어진다 → 알려진 대응 카테고리로 연결. (식별 불가 ≠ 대응 불가)

### L3. 대응 — 효과 기반 등급 대응 (가용성 인식)

**대응 사다리(최소침습 우선):** ① 모니터링↑ → ② 선택적 격리 → ③ 페일세이프(INS/안전모드/채널전환) → ④ (최후)복원·재선출.

효과 버킷별 *기본 복구 깊이*(B19 차등복구 연계):

| 효과 버킷 | 1차 대응 | 복구 깊이 |
|-----------|----------|-----------|
| 항법 이상 | GPS 가중치↓ → INS 추측항법 | 상태추정 복원(deep) |
| 통신 이상 | 채널/주파수 전환, 레이트리밋 | 경로 전환(shallow) |
| 제어 이상 | 위험 명령 action-masking | 안전속도 집행 |
| 노드 이상 | 의심 노드 격리 → 합의 재선출 | 합의 재구성 |
| 인지 이상 | 센서 가중 재조정·프레임 차단 | 전처리 복원 |

> **가용성 가드:** 신뢰도가 낮을수록(=미지일수록) *더 보수적·저침습* 대응을 택한다. "확실하지 않으면 죽이지 않는다" → 과잉대응 페널티 회피.

### L4. 적응 — 미지(unknown) → 기지(known) 전환

- 변조방지(append-only) 로그 → 사후 분석 → **패턴 라이브러리에 추가** → (오프라인) 모델 미세조정/룰 추가 → self-play 재학습.
- 본선 *라운드 간*(또는 매치 내) 적응으로, 같은 공격 재발 시 즉시 기지 대응으로 처리.

### 횡단 — 안전 가드(hard shield) + 범용성

- **Hard shield:** 어떤 정책·어떤 공격에도 *위반 불가*한 불변 제약 — green(정상 임무) 차단 금지, 비행 중 핵심 호스트 재시작 금지, 물리 안전 한계 준수. → **진짜로 처음 보는 공격의 최후 안전망.**
- **범용성(본선 action이 달라도):**
  - 관측/행동을 *의미 단위*로 추상화(효과 버킷 ↔ 대응 카테고리) → 행동 집합이 바뀌어도 매핑은 유지.
  - **GNN 노드 함수**(C8): 노드=드론/호스트, 행동을 노드 단위 함수로 → 토폴로지·군집 크기 변화에 강함(이식성). ⚠️ 단 *성능* 우위는 아님(CC4서 GNN −193 < 휴리스틱 −118) → 범용성 보조용, 성능 주력은 휴리스틱+PPO.

## 3. 에이전트 데이터 흐름 (OODA 매핑)

```
 Observe → L1 불변식 모니터(물리·프로토콜·행동·OOD)
 Orient  → L2 효과 버킷 분류 + 불확실성 추정
 Decide  → L3 등급 대응 선택 (효과 깊이 × 가용성 가중)
 Act     → 대응 실행(격리/INS/마스킹/재선출)
 (루프)  → L4 로깅·오프라인 적응
```

- **LLM은 오프라인 전용:** 미지 패턴 *사후* 해석·새 룰 제안·incident 설명 담당. 실시간 결정 루프엔 미투입(지연 제약).
- **휴리스틱(action-mask) + PPO + 안전가드**의 하이브리드 코어(C3)는 그대로 — 본 설계는 그 *입력단(탐지·분류)과 적응단*을 미지 공격에 견고하게 만든 것.

## 4. 왜 점수로 연결되나

- **방어 견고성(25pt):** 알려진 A1~A21 + *목록에 없는 공격*까지 효과 기반으로 흡수 → "우리 방어는 처음 보는 공격도 막는다."
- **가용성:** 불확실할수록 보수적 대응 → 과잉대응 자해 방지 → `×가용성`에서 유리.
- **범용성:** 본선 action 변경에도 의미 단위 대응 유지 → 이식성·재현성.
- **공격(30pt)과의 대칭 논리:** 공격은 "정해진 행동, 무한한 조합"으로 *창의*, 방어는 "효과 기반, 무한한 공격 흡수"로 *견고* → 한 쌍의 완결된 서사.

## 5. 코드팀 연결 (구현 시 — 본선 대비)

| 계층 | 레포(Burugi/DAH-2026) 연결 | 비고 |
|------|----------------------------|------|
| L1 탐지 | `fleet.py` 신호(snr/gps_err/link_up) + CybORG 관측에 불변식 검사 추가 → 이상 점수 | `defense.py` detector 확장 |
| L2 분류 | 이상 점수를 5버킷으로 라벨링하는 경량 분류기/룰 | 신규 |
| L3 대응 | `agents.py blue_action()`에 효과→행동 매핑(remove/retake/decoy/block/failsafe) + 가용성 가중 | 기존 정책 확장 |
| L4 적응 | 로그 → 오프라인 재학습 | 예선 범위 밖, 본선 대비 |

> **킬러 실험(보고서 §결과):** *학습에 없던 red 변형*(미지 공격 대용)에 대한 **탐지율 / 가용성 유지율**을 측정 → "일반화 곡선" 그래프 = 견고성·범용성을 수치로 증명.

## ⛳ 관통 원칙 (모든 시나리오에 공통 적용)

1. **AI 에이전트 자율성:** 사람이 키보드를 치는 게 아니라 **AI가 정찰→판단→행동을 자율 전개**(대회 본질).
2. **가용성 인식(Availability-aware):** `점수=(공격+방어)×가용성`. → 공격은 *목표 달성하되 시스템을 죽이지 않는 정밀/스텔스*, 방어는 *위협 차단하되 서비스 유지*. (CAGE CC4의 "green 영향 중심 보상"과 동일 철학)
3. **방산 도메인 현실성:** UAV/UGV 실제 구조(MAVLink·GNSS·SATCOM·ROS2·펌웨어)에 근거 → 임베디드 강점 부각.
4. **위성망 특성 활용:** 지연·간헐적 단절을 *공격은 악용*, *방어는 강인하게 견딤*.
5. **검증된 틀 + 우리 혁신:** CAGE/CybORG의 공방 정식화·우승 패턴 위에 LLM 추론·가용성 보상·sim-to-real을 얹는다.

# PART A. 공격 시나리오 (① 30pt — 창의성 + 기술적 완성도)

## A0. 공격 설계 원칙
- **단계적 킬체인**(MITRE ATT&CK 매핑) + **AI 에이전트의 적응적 실행**(상황 따라 경로 선택·회피).
- 점수 극대화 = **공격 성공 + 가용성 유지**(완전 파괴는 가용성 곱셈으로 자해) → **스텔스·지속(persistence)** 지향.

---

## A1. ★공급망 펌웨어 트로이목마 → 군집 웜 확산  *(대표 시나리오)*
- **근거:** UAV는 공급망(부품·펌웨어·OTA) 신뢰가 핵심인데 검증이 취약. CAGE **CC3가 바로 이 위협 모델(드론 군집 + 펌웨어/하드웨어 트로이목마 + 웜)로 검증**됨 → 방산 현실성 + 벤치마크 권위 동시 확보.
- **메커니즘:** 펌웨어/부트로더에 휴면 악성코드 삽입 → 트리거(시간/명령/위치) 시 활성화 → MANET을 통해 **인접 드론으로 웜 확산** → 정보 탈취 또는 허위 텔레메트리 주입.
- **논리적 전개(ATT&CK):** `Supply Chain Compromise(T1195)` → `Firmware 지속(T1542)` → `횡적 이동(군집 전파)` → `데이터 조작(T1565)` / `정보수집`.
- **AI 공격 에이전트 적용:** 에이전트가 **확산 타이밍·대상 드론을 자율 선택**(탐지 회피 위해 점진 확산), 단절 구간을 노려 활성화.
- **가용성 고려:** 즉시 추락시키지 않고 **은밀 잠복 + 선택적 정보 유출** → 가용성 점수 유지하며 공격 점수 획득. (가장 "교과서적" 고득점형)

## A2. GCS 침투 → MAVLink 명령 주입 → 기체 탈취
- **근거:** GCS(지상국)는 보통 일반 IT(PC/네트워크)라 침투가 상대적으로 쉽고, 장악 시 **합법 채널로 명령을 내릴 수 있어** 탐지가 어렵다. MAVLink 1.0 평문 취약성과 결합.
- **메커니즘:** GCS PC 침투(피싱/취약점) → MAVLink 세션 장악 → `COMMAND_LONG` 위조로 항로 변경/무장/RTL 강제 → 기체 탈취 또는 임무 무력화.
- **전개(ATT&CK):** `Initial Access(T1190/피싱)` → `GCS 제어` → `명령 주입/위조(T1565)` → `Impact(T1489 임무 거부)`.
- **AI 적용:** 에이전트가 **정상 운용 패턴을 학습**해 그 틈에 위조 명령을 섞어 IDS 회피(behavior blending).
- **가용성:** 미세 항로 변경 등 **저강도 지속 조작**으로 SLA 유지하며 임무 효과만 훼손.

## A3. GPS 스푸핑 → 항법 기만 → 경로 이탈 / 지오펜스 우회
- **근거:** 민간 GNSS는 비암호화 → 스푸핑 고전적·강력. 물리적 납치(경로 유도)까지 가능해 방산 위협의 대표.
- **메커니즘:** 가짜 GNSS 신호로 위치/시간 기만 → 기체가 잘못된 위치로 비행 → 지오펜스 우회·유도 착륙·경로 이탈.
- **전개:** 신호 정찰 → 점진 오프셋 스푸핑(급격하면 INS 불일치로 탐지) → 목표 좌표로 유도.
- **AI 적용:** 에이전트가 **INS 교차검증을 회피하도록 오프셋 증가율을 동적 조절**(탐지 임계 밑으로).
- **가용성:** 기체는 계속 비행(가용성 유지) → 임무 위치/효과만 왜곡 → 공격 점수 + 가용성 동시 확보.

## A4. 위성링크(SATCOM) 공격 — 지연/단절 악용 + C2 MITM·재전송
- **근거:** **대회 환경이 위성망 모사** → 이 시나리오가 가장 "대회 맞춤". 위성 링크의 고지연·간헐 단절은 고유 공격면.
- **메커니즘:** C2 위성링크에 **MITM/재전송(replay)** → 과거 유효 명령 재생, 또는 단절 구간에 명령 위조 삽입. 링크 미암호화/약한 인증 가정.
- **전개:** 링크 트래픽 정찰 → 재전송 가능 메시지 수집 → 단절·핸드오버 타이밍에 주입.
- **AI 적용:** 에이전트가 **단절 패턴을 예측**해 "관제 사각" 구간에 자율 행동(사람 개입 어려운 시점 공략).
- **가용성:** 링크 자체는 살려두고 **명령 무결성만 훼손** → 가용성 곱셈 방어.

## A5. 컴패니언 컴퓨터 / ROS2(DDS) 미들웨어 익스플로잇
- **근거:** AI·영상처리용 컴패니언(Jetson 등)은 ROS2/DDS 사용. SROS2 미적용 시 토픽 인젝션·노드 탈취 용이. UGV/고급 UAV에 직접 해당.
- **메커니즘:** DDS 디스커버리 악용 → 악성 노드 합류 → 센서/명령 토픽에 위조 데이터 주입 → 자율 판단 왜곡(페이로드·항법).
- **전개(ATT&CK):** `원격 서비스 익스플로잇(T1210)` → `토픽 인젝션(데이터 조작 T1565)` → 자율 스택 오작동.
- **AI 적용:** 에이전트가 어떤 토픽을 노릴지(최대 효과/최소 노출) 자율 선택.
- **가용성:** 핵심 제어 토픽이 아닌 **인지/판단 토픽을 미세 왜곡** → 시스템 동작 유지하며 임무 품질 저하.

## A6. 센서 스푸핑 (UGV LiDAR/카메라/IMU)
- **근거:** UGV는 센서 의존도 극대 → 센서 기만이 치명적. 적대적 예제(카메라) + LiDAR 스푸핑.
- **메커니즘:** 가짜 장애물/유령 물체 주입(LiDAR), 표지판/객체 오인식(카메라 adversarial), IMU 진동 주입.
- **전개:** 센서 입력 조작 → SLAM/인식 오류 → 정지·우회·충돌 유도.
- **AI 적용:** 에이전트가 환경에 맞춰 **가장 효과적인 스푸핑 패턴 생성**(상황 적응).
- **가용성:** 차량 정지(가용성↓) vs 미세 경로 왜곡(가용성 유지) 중 **점수 최적점**을 에이전트가 선택.

## A+ 확장 공격 아이디어 (다양성 강화)

> 제출본이 아니므로 sim 구현 여부와 무관하게 **폭넓게** 수록.

- **A7. RF 재밍 / 미코닝(meaconing) [sim✅ jam]** — 근거: 전자전(EW)에서 가장 흔한 실전 위협. 메커니즘: 광대역·반응형 재밍으로 통신 두절(DoS), 미코닝=정상 신호 재방송 기만. 위성망 단절과 결합.
- **A8. C2 링크 탈취 / 디오쓰(deauth) [개념]** — 근거: 상용 드론 무선 탈취 공개 데모 다수. 메커니즘: 텔레메트리 무선 디오쓰 → 세션 가로채기 → 조종권 탈취.
- **A9. OTA 펌웨어 업데이트 악용 [개념, A1 변형]** — 근거: 공급망 공격의 운영단계 연장. 메커니즘: 위조 펌웨어 푸시·다운그레이드(롤백)로 취약 버전 강제.
- **A10. 스웜 특화 — Sybil·비잔틴 교란 [개념, MANET]** — 근거: 군집 합의·라우팅의 구조적 취약. 메커니즘: 가짜 ID 다수 생성 → 합의 왜곡·블랙홀/웜홀 라우팅 → 군집 분열.
- **A11. 방어 AI 자체 공격 (Adversarial ML) [개념, 차별적]** — 근거: 우리가 AI 방어를 쓰면 그 AI가 표적(메타 위협). 메커니즘: 탐지 회피(evasion)·모델 포이즈닝·적대적 텔레메트리로 오탐 유발. → ②의 한계를 스스로 짚는 고급 논점.
- **A12. 임베디드 사이드채널 [개념]** — 근거: 물리 접근 가능한 무인체계의 임베디드 특성. 메커니즘: 전력분석·타이밍으로 키/로직 추출, 글리치 공격.
- **A13. PNT/시간동기 공격 [개념, 위성망 특화]** — 근거: 위성망·항법은 GPS 시간동기 의존. 메커니즘: 시간 스푸핑으로 타임스탬프·로그·재전송 방어 교란.
- **A14. 다영역 협조 공격 [sim 일부✅]** — 근거: 실전 공격은 동시다발. 메커니즘: 재밍(은폐)+웜(침투)+GPS(유도)를 시간 조율 → 단일 방어로 못 막게.
- **A15. 내부자 위협(Insider) [개념]** — 근거: 운용자·정비 인력의 합법 접근 악용(탐지 최난). 메커니즘: 정상 권한으로 펌웨어·미션 설정 변조·정보 유출.
- **A16. ISR 임무데이터 탈취(Exfiltration) [개념]** — 근거: 무인체계의 핵심 가치 = 수집 정보. 메커니즘: 페이로드 영상/표적 데이터를 저속 채널·스테가노그래피로 은밀 유출.
- **A17. 스웜 지휘체계 탈취(C2 takeover) [개념, A10 상위]** — 근거: 군집 조정/리더 노드 장악 시 편대 전체 통제. 메커니즘: 리더 선출·조정 노드 침해 → 편대 재지정·임무 변경·자폭 유도.

# PART B. 방어 시나리오 (② 25pt — 견고성 + 실현 가능성)

## B0. 방어 설계 원칙
- **계층 방어(Defense-in-Depth):** 탐지 + 대응 + 복원을 여러 층에.
- **가용성 보존형 대응:** 차단보다 **격리·점진 대응·페일세이프** 우선(과잉 차단 = 자해).
- **공격 시나리오 A와 1:1 매핑**으로 "막을 수 있음"을 입증(견고성).

## B1. 텔레메트리 / 행위 이상탐지 (시계열 ML)  ← A2,A3,A4
- **근거:** 위조 명령·스푸핑은 **정상 패턴에서 미세 이탈**을 남김. 임베디드에서 수집되는 텔레메트리(자세·고도·전류·링크품질)는 시계열 이상탐지의 좋은 입력.
- **메커니즘:** 경량 시계열 모델(예: LSTM-AE, Isolation Forest) + 규칙으로 이상 점수화 → 임계 초과 시 경보.
- **가용성 영향:** 탐지는 비파괴적 → 가용성 무손실. 오탐 억제가 관건(green 노이즈 모델로 보정).

## B2. GPS 스푸핑 탐지 (센서 퓨전 교차검증 + RAIM)  ← A3
- **근거:** §5 도메인 — INS와 GNSS의 **물리적 일관성**을 깨면 스푸핑 탐지 가능. RAIM·다중 GNSS·도래각.
- **메커니즘:** INS 추정 위치 vs GNSS 위치 잔차 모니터링 + 신호 파워/도래각 이상 + RAIM. 불일치 시 **INS 기반 추측항법으로 페일세이프**.
- **가용성:** GNSS 의심 시에도 **INS로 비행 유지** → 가용성 보존하며 공격 무력화(핵심 우승 포인트).

## B3. MAVLink 무결성 / 인증  ← A2,A4
- **근거:** 평문 MAVLink가 주입/재전송의 근본 원인 → **서명·시퀀스·논스**로 차단.
- **메커니즘:** MAVLink 2.0 **메시지 서명** 활성화, 메시지 카운터/타임스탬프로 **재전송 거부**, 명령 화이트리스트·레이트리밋, 비정상 명령 시퀀스 탐지.
- **가용성:** 위조만 거부하고 정상 명령은 통과 → 가용성 영향 최소.

## B4. 네트워크 분할 + IDS + 존 트래픽 제어  ← A2,A5
- **근거:** CAGE **CC4의 BlockTrafficZone/AllowTrafficZone**(존 단위 방화벽 제어)이 검증된 방어 행동. 횡적 이동·확산 차단.
- **메커니즘:** GCS·컴패니언·FCU를 존으로 분할, IDS로 이상 플로우 탐지, **의심 존만 선택 차단**(전면 차단 ✗).
- **가용성:** 핵심 제어 경로는 유지하고 의심 경로만 차단 → 가용성 보존형.

## B5. ★Canaries & Whistles — 군집 감염 탐지·전파  ← A1
- **근거:** **CC3 우승작**(휴리스틱이 PPO를 ~5배 앞섬). A1(군집 웜)에 대한 직접 대응책으로 강력하고 **구현 단순**.
- **메커니즘:** 각 정상 드론이 주기적 **ID 하트비트(Canary)** 브로드캐스트 → 멈추면 감염 추정. 감염 ID를 **군집 전체로 재전파(Whistle)** → 이웃이 해당 노드 격리/불신.
- **가용성:** 감염 의심 노드만 격리, 군집 임무는 지속 → 가용성 유지하며 확산 차단. **고득점·고현실성·저구현비용**의 삼박자.

## B6. 기만(Deception) — 디코이 / 허니팟  ← A1,A2,A5
- **근거:** CAGE **CC2 우승 레시피 = PPO + 탐욕적 디코이**. 공격자를 가짜로 유인해 **조기 탐지 + 시간 소모**.
- **메커니즘:** 가짜 서비스/가짜 드론/허니 토픽 배치 → 공격자가 건드리면(정상 사용자는 안 건드림) **명백한 침해 신호**.
- **가용성:** 디코이는 실제 서비스와 분리 → 가용성 무손실 + 탐지력↑.

## B7. 가용성 보존형 자율 대응 (대응 정책의 핵심)
- **근거:** `×가용성` 채점의 직접 대응. CAGE의 **Restore(복원)도 페널티**(운영중단 비용)라는 교훈 → "끄고 재시작"은 최후수단.
- **메커니즘:** 대응 단계화 — ①모니터링 강화 → ②격리/세분 차단 → ③페일세이프(INS항법·안전모드) → ④(최후)복원/재시작. 에이전트가 **위협 심각도 vs 가용성 손실**을 저울질해 최소침습 대응 선택.
- **가용성:** 설계 자체가 가용성 최적화 → 본선 점수 직결.

# PART C. AI 에이전트 아키텍처 (③ 25pt — 구체성 + 혁신성)  ★차별화 무기

## C0. 설계 철학
- **3-에이전트 정식화(검증된 틀):** Blue(방어)/Red(공격)/**Green(정상 임무 트래픽·운용=가용성 기준)**. ※ UAV/UGV엔 사람 "사용자"가 없으므로 CybORG의 green을 **드론의 정상 통신·임무 수행**으로 재정의(가용성 측정 기준).
- **하이브리드 코어(혁신+실현):** **도메인 휴리스틱 + RL(PPO) + LLM 추론**. CAGE 교훈 = 순수 딥RL/MARL보다 **휴리스틱·하이브리드가 우월**.

## C2. 정식화 (관측 · 행동 · 보상)
- **관측(Observation, POMDP):** 텔레메트리(자세·고도·링크품질), 네트워크 플로우, 시스템/ROS2 로그, IDS 알람, GNSS/INS 잔차. → 정규화·비트벡터/특징벡터화(CybORG `BlueTableWrapper` 방식 참조).
- **행동(Action):**
  - Blue: `Monitor / Analyse / Isolate(격리) / Restore(복원) / DeployDecoy / BlockZone / Failsafe(INS전환)`.
  - Red: `Recon / Exploit / Inject / Propagate / Persist / Impact`.
- **보상(Reward) — 가용성 인식형 (핵심 혁신):**
  ```
  R_blue = Σ(위협차단 가점) − Σ(침해 손실) − λ·(가용성 손실)   # 과잉대응·다운 시 큰 페널티
  R_red  = Σ(공격 목표 달성) − μ·(노출·탐지 위험)             # ⚠️곱셈 금지→가중합(검증반영)
  ```
  → 대회 `(공격+방어)×가용성`을 학습 신호로 내재화. (CAGE의 "green 영향 중심" 보상 일반화)

## C3. 의사결정 코어 — 하이브리드 (왜 이렇게?)
- **(a) 도메인 휴리스틱:** Canary/Whistle, RAIM 임계, 명령 화이트리스트 등 **검증된 규칙**. → 즉시 동작·해석가능·경량(임베디드 적합). *CAGE에서 휴리스틱이 RL을 이긴 근거.*
- **(b) RL(PPO):** 휴리스틱으로 안 잡히는 미묘한 상황의 정책 최적화. **DQN 회피(부분관측·노이즈 불안정), Stable-Baselines3 사용.**
- **(c) LLM 에이전틱 추론:** **novel·복합 상황 해석, 다단계 계획, 도구 선택**(예: "이 알람 조합은 A4 재전송 의심 → B3 활성화"). 느린 고수준 판단 담당.
- **(d) 안전 가드(Safety Guard):** 어떤 정책도 **가용성·안전 제약 위반 행동은 차단**(예: 비행 중 전체 재시작 금지). 계층형 = 상위 LLM 플래너 + 하위 휴리스틱/RL 실행기.

## C4. 학습 전략
- **자가대전(Self-play):** Blue↔Red 동시 강화로 강건성.
- **커리큘럼:** 단순 공격(A2)→복합(A1 군집)으로 난이도 상승.
- **Green 시뮬레이션:** 정상 트래픽 주입으로 오탐 억제·가용성 측정 현실화.
- **sim-to-real 고려:** 임베디드 자원/지연 제약을 학습에 반영 → 실제 이식성(차별점).

## C5. 혁신 포인트 (보고서 강조)
1. **가용성 인식 보상함수** — 대회 채점을 에이전트 목표로 내재화(직접적·논리적).
2. **휴리스틱+RL+LLM 3중 하이브리드** — CAGE 교훈 기반, 해석가능성+적응성+계획성.
3. **임베디드 sim-to-real** — "게임을 넘어 실전"(CAGE의 공인된 한계를 우리가 메움).
4. **Blue/Red 공용 골격 + self-play** — 공격·방어를 한 프레임워크로(설계 일관성).

## C+ 대안 / 다양화 아키텍처 아이디어

> 핵심 설계(C1~C7)는 휴리스틱+PPO+LLM 하이브리드. 아래는 보고서 다양성용 대안·변형.

- **C8. GNN 기반 정책(그래프 표현)** — 근거: 군집/네트워크는 본질적으로 그래프. 노드=드론/호스트·엣지=링크, 행동을 노드 단위 함수로 → **네트워크 크기 자동 확장·미관측 토폴로지 zero-shot**(Cybermonic GNN+PPO). 무작위 군집 구성에 강함.
- **C9. 계층형 MARL(CTDE)** — 근거: CC3는 드론당 blue 1개(분산 방어). 중앙학습·분산실행(MAPPO) + 상위 조정자. 단 무작위 초기화 적응·파라미터 공유 병목 주의(검증 교훈).
- **C10. Decision Transformer / 시퀀스 모델** — 근거: 오프라인 RL·임베디드 경량. 과거 (관측·행동·보상) 시퀀스로 행동 예측 → 자원 제약 유리.
- **C11. 앙상블 정책** — 근거: 단일 정책 과적합 회피, CAGE에서 앙상블이 일반화 우수. 다수 정책 투표/선택.
- **C12. LLM-as-Judge / 설명가능 계층** — 근거: 사람이 읽는 incident 판단·심사 보조. 추론루프 밖 사후 설명(데모·보고서 가치).
- **C13. 월드모델 기반(model-based RL)** — 근거: 동역학 학습으로 샘플효율↑·위협 전개 선제 예측.
- **C14. 뉴로심볼릭(규칙+학습)** — 근거: 도메인 규칙(MAVLink 검증·RAIM)은 심볼릭, 미세조정은 학습 → 해석가능성+적응성.

## ✅ PART C 검증 반영 (근거: `AI아키텍처_검증.md`, 2023~25 SOTA 교차검증)

- ✅ **Blue/Red/Green + 가용성 보상 = CAGE CC4 표준과 동일** → "발명"이 아닌 **"검증된 표준 채택"**으로 어필.
- ✅ **R_blue(덧셈 + λ·가용성손실) 타당.** 단 셰이핑 항은 **희소·최소화**(과다 셰이핑 = reward hacking 위험).
- ⚠️ **R_red 정정(중요):** 곱셈 `×availability_factor` → **가중합으로 분해**: `R_red = Σ(공격목표) − μ·(노출·탐지 위험)`. *가용성 곱셈은 대회 "채점식"에만, 에이전트 "보상"은 덧셈으로.* (근거 arXiv:2604.08805 — 곱셈은 그래디언트 소실/폭주·reward hacking)
- 🟡 **LLM은 추론루프에서 빼고 "오프라인"으로** — ① 보상설계(Eureka식) ② PPO 부트스트랩(action prior를 **확률분포로**; 단일행동 추천 금지) ③ incident 설명·데모. (8B=3.69s/step → 임베디드 실시간 불가, arXiv:2509.05311 / 순수 LLM 방어는 장기 누적보상서 RL에 밀림 2505.04843)
- 🟡 **하이브리드 = "계층 분리":** 휴리스틱=**action-masking**(위험·불가 행동 사전 제거), RL=PPO 정책, LLM=상위 계획(오프라인), 가드=**hard shield**(green 차단 금지·임무 중 핵심호스트 재시작 제한). SMT/Lyapunov 형식검증은 과함.
- 🟡 **Red는 FSM+ATT&CK 스크립트형 유지**(자율 LLM red 성공률 ~31%·무한루프 → 재현성 우선).
- 🟡 **환경은 CC4 / Cyberwheel / CybORG++ 채택**(직접 구현 금지 — 시간 낭비).
- 📊 **평가지표:** 보상 외 **MTTD/MTTR · 가용성 유지율 · 침해 호스트 수** + 다중 시드.

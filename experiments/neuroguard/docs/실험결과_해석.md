## 결과 1 — 시나리오별 탐지·보정 (seeds 평균)

| 시나리오 | 탐지기 | 대응 | jam F1 | gps F1 | gpsErr 전 | gpsErr 후 |
|----------|--------|------|:---:|:---:|:---:|:---:|
| baseline | none | none | 0.0 | 0.0 | 0.0 | 0.0 |
| combined | none | none | 0.0 | 0.0 | 89.1 | 89.1 |
| **defended** | **multisensor** | **safe_mode** | **1.0** | **0.96** | **89.1** | **8.5** |

**해석:**
- **defended(멀티센서+safe_mode)** 가 재밍·GPS 모두 탐지(F1 1.0/0.96) + **위치오차 89.1m → 8.5m로 복원** → B2(센서퓨전 교차검증 + 가용성 보존 대응) 실측 입증.
- **fsm_red(단일센서 threshold)** 는 재밍은 잡지만 **GPS F1 = 0** → "단일 센서로는 GPS 스푸핑 못 잡는다, 멀티센서 교차검증 필요"라는 우리 B1/B2 설계 논거를 실측 확인.
- 무방어(baseline/combined/passive)는 보정 없음(89m 오차 그대로).

## 결과 2 — ★은밀 GPS 스푸핑이 탐지를 회피 (검토 C1 실측 입증) · `fig4`

| drift(m/step) | 탐지 recall | 최대 gps_err | 비고 |
|:---:|:---:|:---:|------|
| 0.2 | **0.00** | 5.9 | 완전 회피 |
| 0.4 | **0.39** | 11.9 | 회피 |
| 0.6 | 0.60 | 17.8 | |
| 1.0 | 0.75 | 29.7 | |
| 2.0 | 0.88 | 59.4 | |
| 4.0 | 0.95 | 118.8 | 즉시 탐지 |

**해석 (가장 중요):**
- **저드리프트(≤0.4m/step) GPS 스푸핑은 멀티센서 임계(gps_thresh=8)를 넘지 않아 탐지를 회피**(recall 0~0.39). → 우리 검토의 **C1 "탐지↔은밀 공격 모순"(A19 Raven·A3-slow가 L1 임계를 설계상 회피)을 실측으로 증명.**
- 동시에 **은밀 ↔ 효과 trade-off**가 정량화됨: 회피하려면 오차도 작아야 함(drift 0.2 = 최대 5.9m, 거의 무해). 큰 피해(89m)를 내려면 drift↑ → recall 0.93으로 잡힘.
- **시사점:** 단일 임계(L1)는 "저드리프트 회피 영역"이 존재 → **L2 교차모달(다중센서·VLM 일관성)·합의(B5/B9) 다층 방어로 그 영역을 메워야** 한다(B20 다층 설계의 실측 근거).

| 공격↓ \ 방어→ | rule | llm | rl |

(셀 = 최종감염비율 \| blue보상) → **휴리스틱(rule) 방어가 전 공격자에 최저 감염+최고 보상. 멀티에이전트 RL 최악.** RL은 탐지 F1 0.87로 높지만 봉쇄 실패 = **탐지 ≠ 방어**. CAGE CC4·우리 검토와 정합.

**(B) 단일 vs 멀티 vs 휴리스틱** (우리 추가, vs rule-red, held-out seeds 1000~1004) · `fig8`,`fig9`

| 방어자 | 최종 감염 | 보상 |
|---|:---:|:---:|
| **heuristic (multi)** | **0.356** | **−453** |
| RL-multi (드론별 공유 Q) | 0.867 | −534 |
| RL-single (중앙 글로벌 Q) | 0.756 | −594 |

→ **휴리스틱 압도.** "멀티가 무조건 유리"는 거짓 — 봉쇄(감염)는 **RL-single(0.76) > RL-multi(0.87)**, 보상은 반대 → trade-off, **둘 다 휴리스틱에 패배**. `fig9`: 휴리스틱만 웜을 평탄 봉쇄(~6), RL 둘 다 광범위 전파(13~16).
⚠️ 약한 tabular-Q 기준. 강한 deep RL(PPO/GNN)은 결과 달라질 수 있음 → 7.4 로드맵.
